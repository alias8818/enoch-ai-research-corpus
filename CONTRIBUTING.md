# Contributing

Thanks for taking a look. This project is being released for inspection, critique, and replication.

## Ground rules

- Do not submit secrets, local tokens, private IPs, private logs, private run state, or personal/career notes.
- Keep generated-research framing explicit: corpus papers are AI-generated artifacts, not human-authored or peer-reviewed papers.
- Prefer small, reviewable changes with tests or quality-scan evidence.
- For security concerns, do not open a public issue; follow `SECURITY.md`.

## Development flow

1. Fork or branch.
2. Make a focused change.
3. Run the documented checks.
4. Open a PR with evidence in the template.
