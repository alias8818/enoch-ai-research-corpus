### Key run-note evidence

- - Budget 320: gate chose colorized for all rows and matched colorized gains: field recall 0.5417 vs vanilla 0.0000, mean tokens 198.69 vs 320.
- - Budget 480: gate chose vanilla for all rows and eliminated the fixed-colorization regression: field recall 0.8333 and evidence recall 0.7917, equal to vanilla, while fixed colorized fell to field recall 0.625 and evidence recall 0.5938.
- SQuAD / extractive QA pilots:
- - Late-answer stress budget 160: gate chose colorized for all 50 tasks and matched fixed colorized, beating vanilla by +65.62 F1 points, +68 citation points, +84 kept-evidence recall points, with 12.55% fewer prompt tokens.
- - Retrieval budget 160: gate mixed 15 colorized / 35 vanilla and beat vanilla by +4.30 F1 points, +6 citation points, +8 kept-evidence recall points; it also beat fixed colorized by +1.02 F1 points but used more tokens (153.28 vs 139.74) because it preserved vanilla when raw looked sufficient.
- - Retrieval budget 220: gate mostly chose vanilla (46 vanilla / 4 colorized), tying vanilla F1/citation/evidence and avoiding fixed colorized's evidence-recall loss (-6 points) while giving up fixed colorized's small +0.67 F1 pilot gain. Parent 80-example run showed colorized losing at budget 220, so this conservative route is acceptable but should be viewed as a precision-over-token-savings gate.
- Throughput/resource notes:
- - SQuAD runs used the cached local DistilBERT extractive QA model offline; no helper server was left running.
- - Pilot throughput: 18.2-23.1 model calls/sec wall across the SQuAD runs; process RSS/PSS roughly 1.0 GiB; MemAvailable roughly 120.9 GiB; SwapFree 0.
- - Technical-doc benchmark is CPU-only prompt packing and completed each 80-pack sweep in ~0.10-0.12 s. Gate construction latency is ~2.1-2.7 ms because it builds both candidate prompts for diagnostic simplicity; this can be optimized later by computing only features first and lazily building the chosen prompt.
- ## Interpretation
- The adaptive boundary-colorization gate materially advances the branch hypothesis. It preserves the strong tight/late-evidence wins from fixed colorization and prevents the known 480-token technical-doc/paraphrase regression by falling back to vanilla when raw prompt cues indicate the answer contract is already reachable. The main limitation is conservative routing at mid-budget open-domain retrieval: it may leave small token-saving or F1 wins on the table, but it avoids the more important failure mode of destructive colorized selection when vanilla is already adequate.
