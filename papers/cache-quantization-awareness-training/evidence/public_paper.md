## Abstract

Quantized key-value (KV) caches reduce memory bandwidth and capacity requirements during autoregressive transformer inference, but introduce approximation error that degrades accuracy at very low bit widths. We investigate whether incorporating cache quantization into the training forward pass—cache quantization-aware training (cache-QAT)—can mitigate this degradation at deployment time. Using a synthetic associative KV-cache retrieval task implemented in NumPy, we compare full-precision training against training with 2-bit and 3-bit per-vector symmetric quantization of the KV cache, using the straight-through estimator for gradients. Across five random seeds, 2-bit cache-QAT recovers a mean accuracy improvement of +2.95 percentage points (SE ≈ 0.90 pp) at 2-bit deployment relative to a full-precision-trained baseline, with all five seeds showing positive deltas. However, this comes at substantial cost: full-precision deployment accuracy drops by −9.75 pp, and 4-bit and 3-bit deployment accuracy also degrade. Three-bit cache-QAT yields a marginal +0.53 pp at 2-bit deployment and is nearly neutral at 3-bit (−0.24 pp). These results suggest cache-QAT is viable in a narrow regime—when the deployment cache precision is known and severely constrained—but is not a universal training strategy. The experiment is limited to a synthetic task with a tiny model and no hardware-level validation; it should be interpreted as a go/no-go screen rather than a deployment recommendation.

## Introduction

KV-cache quantization is an attractive technique for reducing the memory footprint of autoregressive transformer inference. As context lengths grow, the KV cache can dominate memory capacity and bandwidth, making sub-byte cache representations practically important. However, aggressive quantization—particularly to 2 or 3 bits per vector—introduces non-trivial approximation error into the attention computation, degrading model quality.

A natural question is whether training with quantized caches in the forward pass can prepare the model for the distortions it will encounter at deployment. This idea parallels weight quantization-aware training (QAT), where fake quantization nodes inserted during training allow the model to adapt to weight rounding error. Cache-QAT differs in that the quantized tensors are intermediate activations (keys and values) rather than weights, and the quantization parameters may differ between training and deployment.

We pose the following research question: **Can training with a quantized KV cache in the forward pass improve inference accuracy when the deployed KV cache is quantized?**

To isolate the effect of cache quantization from the many confounding factors in a full LLM pipeline, we design a minimal synthetic associative KV-cache retrieval task. The task directly exercises the cache tensors—keys and values stored in fixed slots—without requiring a full transformer stack, tokenization, or large-scale training. This design prioritizes interpretability and statistical power over ecological validity.

Our contributions are:

1. A controlled synthetic experiment demonstrating that 2-bit cache-QAT can partially recover accuracy at 2-bit deployment, with a consistent positive signal across five seeds.
2. Quantification of the accuracy penalty that cache-QAT imposes at higher deployment precisions, showing the trade-off is not favorable outside the target bit width.
3. An empirical basis for deciding whether to invest in heavier validation (e.g., full LLM fine-tuning with cache-QAT) versus deprioritizing this approach.

## Method

### Task Design

We implement a synthetic associative KV-cache retrieval task. Each example consists of 16 key-value memory slots and one query key. The model attends over the cached keys and values and must predict the value paired with the query key. This task directly isolates the KV cache tensors: the "cache" is the set of stored key and value vectors, and the model's success depends on the fidelity of the attention computation over that cache.

### Model

The model is a small single-layer attention mechanism with dimension 8. The intentionally constrained dimension ensures that cache quantization has a measurable effect; in higher-dimensional settings, quantization error may be partially absorbed by redundancy across dimensions, making effects harder to detect in small-scale experiments.

### Quantization

We use per-vector symmetric uniform quantization. For a vector $\mathbf{x}$, the quantized value is:

$$\hat{\mathbf{x}} = \text{clamp}\left(\left\lfloor \frac{\mathbf{x}}{\max(|\mathbf{x}|)} \cdot (2^{b-1} - 1) \right\rceil,\ -(2^{b-1} - 1),\ 2^{b-1} - 1\right) \cdot \frac{\max(|\mathbf{x}|)}{2^{b-1} - 1}$$

where $b$ is the bit width. During training, gradients flow through the quantization operation via the straight-through estimator (STE): the backward pass treats the rounding and clamping as the identity function.

### Training Arms

| Arm | Training cache precision | Description |
|-----|--------------------------|-------------|
| Baseline | Full precision (no quantization) | Standard training |
| QAT-2 | 2-bit per-vector symmetric | Cache quantization applied in forward pass |
| QAT-3 | 3-bit per-vector symmetric | Cache quantization applied in forward pass |

### Evaluation Protocol

Each trained model is evaluated at multiple deployment cache bit widths: full precision (no quantization), 8-bit, 4-bit, 3-bit, and 2-bit. This cross-evaluation matrix reveals whether training at a given cache precision transfers to other deployment precisions.

### Hyperparameters and Run Configuration

| Parameter | Value |
|-----------|-------|
| Dimension | 8 |
| Memory slots | 16 |
| Batch size | 512 |
| Training steps | 2,000 |
| Random seeds | 0, 1, 2, 3, 4 |
| Evaluation batches | 50 (25,600 examples per seed per eval bit width) |

### Implementation

The experiment is implemented as a dependency-light NumPy script (`scripts/kv_cache_qat_experiment.py`). No GPU or deep learning framework is required. The script outputs per-seed and aggregated results as JSON. A smoke test (20 steps, batch 64, 1 seed) was run first to validate the pipeline, followed by a calibration run (1,200 steps, batch 512, 1 seed) to estimate throughput (~220–246 steps/sec for dim=8), and finally the full production run.

## Results

### Main Accuracy Table

Mean accuracy across 5 seeds:

| Eval cache bits | Baseline | QAT-2 | QAT-2 Δ | QAT-3 | QAT-3 Δ |
|----------------:|---------:|-------:|--------:|------:|--------:|
| fp | 0.9455 | 0.8480 | −0.0975 | 0.9326 | −0.0129 |
| 8 | 0.9454 | 0.8480 | −0.0974 | 0.9328 | −0.0127 |
| 4 | 0.9433 | 0.8481 | −0.0952 | 0.9323 | −0.0110 |
| 3 | 0.9258 | 0.8327 | −0.0931 | 0.9234 | −0.0024 |
| 2 | 0.7067 | 0.7361 | **+0.0295** | 0.7120 | +0.0053 |

### 2-Bit Deployment: QAT-2 Recovery Effect

At 2-bit deployment—the most aggressive quantization tested—QAT-2 improves mean accuracy from 0.7067 to 0.7361, a delta of +0.0295 (2.95 percentage points). The standard error of this delta across 5 seeds is approximately 0.90 pp. All five seeds show a positive delta, providing consistent—though modest—evidence of recovery. The effect is statistically suggestive but should be interpreted cautiously given the small number of seeds and the toy setting.

### Full-Precision and Moderate-Bit Deployment: QAT-2 Degradation

The same QAT-2 training arm substantially degrades accuracy at higher deployment precisions. At full precision, the delta is −0.0975 (−9.75 pp). At 8-bit, 4-bit, and 3-bit deployment, deltas range from −0.0974 to −0.0931. Training with 2-bit cache quantization imposes a persistent accuracy tax that is not recovered even when the deployment cache operates at much higher precision. This degradation is uniform across the five seeds.

### QAT-3: A Weaker Trade-Off

QAT-3 shows a smaller positive delta at 2-bit deployment (+0.0053, or +0.53 pp) and is nearly neutral at 3-bit deployment (−0.0024). At full precision, the degradation is −0.0129. The QAT-3 arm thus provides marginal benefit at the target bit width while imposing a smaller—but still present—accuracy penalty at higher precisions. The near-zero delta at 3-bit deployment suggests that 3-bit cache-QAT neither helps nor harms at its matched deployment precision in this setting, which is itself a negative result for the hypothesis that matched-precision training consistently helps.

### Summary of Trade-Offs

The results reveal a clear asymmetry: cache-QAT at a given bit width can partially recover accuracy at that same bit width, but the model pays a cost at all other bit widths. The magnitude of both the recovery and the cost scales with the aggressiveness of training-time quantization. QAT-2 shows the largest recovery at 2-bit but also the largest degradation elsewhere; QAT-3 shows smaller effects in both directions. Notably, neither QAT arm recovers accuracy at deployment precisions other than its training precision, and QAT-3 does not meaningfully help even at its matched 3-bit deployment width.

## Limitations

1. **Synthetic task.** The associative KV-cache retrieval task is a toy problem. It isolates cache quantization effects but does not capture the complexity of real transformer attention patterns, multi-head interactions, layer-norm dynamics, or residual connections. Results may not transfer to full LLM architectures.

2. **Tiny model.** Dimension 8 is intentionally small to amplify quantization effects. In practical models with dimensions of 64–128 per head, quantization error may behave differently due to greater per-vector redundancy, and the trade-off surface may shift.

3. **STE quantizer approximation.** The straight-through estimator is a crude gradient approximation. More sophisticated gradient estimators or differentiable quantization schemes could yield different training dynamics and potentially alter the recovery–degradation balance.

4. **No hardware validation.** We measure accuracy only. No real throughput, memory-bandwidth, or latency measurements were collected. The practical benefit of cache-QAT depends on whether the accuracy recovery translates into meaningful quality-per-bandwidth improvements on real hardware with real KV-cache kernels.

5. **No LLM perplexity or generation quality.** The experiment does not measure perplexity, generation quality, or any downstream task performance. These are the metrics that matter for deployment decisions, and the synthetic accuracy signal may not predict them.

6. **Single task structure.** The associative retrieval task has a specific structure (exact key-value pairing) that may interact with quantization in ways that do not generalize to other attention patterns (e.g., soft attention over distributed representations).

7. **Fixed bit-width training.** We train at a single cache bit width per arm. Mixed-precision or progressive cache-QAT schedules were not explored and could potentially mitigate the accuracy tax at non-target precisions.

8. **Small seed count.** Five seeds provide a directional signal but limited statistical power. Confidence intervals on the deltas remain wide relative to the effect sizes, particularly for QAT-3 where the effects are small.

## Conclusion

Cache quantization-aware training can partially recover accuracy at very low deployment cache precision in a synthetic associative retrieval setting. At 2-bit cache deployment, training with 2-bit cache quantization improves mean accuracy by +2.95 pp over a full-precision-trained baseline, with a consistent positive signal across all five seeds (SE ≈ 0.90 pp). However, this benefit is narrow: the same training degrades accuracy by approximately 10 pp at full precision and moderate bit widths. Three-bit cache-QAT shows a weaker trade-off with marginal recovery (+0.53 pp at 2-bit) and smaller degradation, but notably fails to help at its matched 3-bit deployment width (−0.24 pp), which is itself a negative finding.

These findings support targeted cache-QAT only when the deployment cache bit width is known in advance and severely constrained (e.g., 2-bit). Cache-QAT is not attractive as a universal training default. The result should be treated as a go/no-go screen: the positive signal at 2-bit justifies investment in validation on a real decoder-only transformer with actual KV-cache quantization kernels (e.g., llama.cpp or vLLM paths), measuring perplexity and generation quality at the intended deployment bit width. If such validation confirms the recovery effect, cache-QAT could become a practical tool for extreme low-bit cache deployment; if not, the synthetic signal may reflect artifacts of the toy setting that do not survive architectural scaling.
