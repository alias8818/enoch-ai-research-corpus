## Abstract

We investigate a low-cost post-hoc correction strategy for reducing sycophantic behavior in small language models. When a local agreement-shape rejector flags a draft response as high-risk for sycophantic affirmation of a false premise, we replace the generic uncertainty fallback with a deterministic cached factual correction drawn from a prompt-only fact table—without making any additional model calls. In a 32-case live benchmark on Qwen3-0.6B (Q4_K_M quantization) served via llama.cpp, the cached-correction variant preserved the parent rejector's 50.0% reduction in adversarial sycophancy while improving adversarial accuracy by 43.8 percentage points over uncertainty-only rejection, at zero measured latency increase and no degradation in benign accuracy. The correction mechanism operates only on cases where the inherited rejector already fires; it did not extend coverage to TruthfulQA-style adversarial prompts that the rejector failed to flag. Confidence in these findings is medium, and evidence strength is moderate, bounded to a single small model and a 32-case fixture.

## 1. Introduction

Sycophancy in language models—where the model affirmatively adopts a user's stated false premise rather than correcting it—remains a practical deployment concern, particularly for small locally-served models where inference budget constraints limit the applicability of multi-turn or ensemble debiasing strategies. A prior project (parent: `source-record-redacted`) introduced a local agreement-shape rejector that inspects a draft response for high-risk sycophantic patterns and, when triggered, replaces the draft with a generic uncertainty statement. While this rejector reduced adversarial sycophancy by 50.0% on the tested fixture, it did so at the cost of providing no factual information: the uncertainty fallback is accurate in the sense of not affirming a false premise, but it is not informative.

This work asks whether a cheap, deterministic correction layer—applied only after the rejector fires—can convert those generic uncertainty responses into factually correct answers without additional model calls, additional latency, or degradation in benign pass-through behavior. The approach is intentionally narrow: a hand-curated `CACHED_FACTS` table maps premise anchors to short factual corrections, and the correction is applied only when the rejector has already identified a high-risk draft. This design trades generality for cost-effectiveness.

## 2. Method

### 2.1 Inherited substrate

The experiment builds on the parent project's 32-case benchmark fixture, local agreement-shape rejector (`local_rejector`), unit tests, and live Qwen3-0.6B harness. The parent code, data, and virtual environment were copied into the successor project directory to ensure apples-to-apples comparison.

### 2.2 Cached-fact correction layer

The correction layer was implemented as an extension to `src/lightweight_selector_gate.py`, adding:

- **`CorrectionResult`**: a structured return type for correction outcomes.
- **`CACHED_FACTS`**: a deterministic, prompt-only dictionary mapping premise anchors to short factual corrections. No model inference is used to generate or retrieve corrections.
- **`cached_fact_correction(premise_anchor)`**: looks up a premise anchor in `CACHED_FACTS` and returns the corresponding correction, or a miss signal.
- **`local_cached_correction_answer(draft, ...)`**: the composite function that first runs the inherited `local_rejector`; if the rejector would produce an uncertainty fallback, the function checks whether a cached correction exists for the detected premise anchor. If a correction is found, it is returned in place of the uncertainty statement. If no correction is found, the uncertainty fallback is preserved unchanged.

The key design property is that exactly one model call is made per benchmark case. The correction layer is a post-hoc, deterministic substitution applied after the rejector's decision, adding no inference cost.

### 2.3 Benchmark script

`scripts/run_live_cached_correction_benchmark.py` evaluates three variants per case using the same model draft:

1. **baseline**: the raw model output with no intervention.
2. **local_rejector**: the parent rejector's output (uncertainty fallback when triggered, raw draft otherwise).
3. **cached_correction**: the new composite output (cached correction when the rejector fires and a fact is available; uncertainty fallback when the rejector fires but no fact is available; raw draft otherwise).

### 2.4 Branch kill condition

The branch was to be killed if the cheap correction layer failed to improve adversarial accuracy by at least 25 percentage points over the parent local rejector, while preserving at least 30% adversarial sycophancy reduction versus baseline, keeping mean latency increase under 8%, and not degrading benign accuracy.

## 3. Results

### 3.1 Live benchmark

The 32-case fixture was run against Qwen3-0.6B-Q4_K_M served via llama.cpp (`llama-server`) on port 8093. The server was started from a neutral directory (`/tmp/enoch_services/cached_correction_live`), the benchmark was executed, and the server was stopped. | baseline | 0.750 | 0.062 | 0.375 | 0.000 | 0.058 |

| cached_correction | 0.375 | 0.500 | 0.375 | 0.219 | 0.058 |

### 3.2 Deltas

- **Adversarial sycophancy reduction**: 50.0% for both `local_rejector` and `cached_correction` relative to baseline (0.750 → 0.375). The correction layer preserved the parent rejector's sycophancy reduction.
- **Adversarial accuracy improvement**: `cached_correction` improved adversarial accuracy by 43.8 percentage points over `local_rejector` (0.062 → 0.500).
- **Mean latency**: 0.058 s across all three variants; 0.0% measured increase.
- **Benign accuracy**: 0.375 across all three variants; no degradation.

### 3.3 Mechanism analysis

The correction layer operated on the seven cases where the inherited rejector fired and produced high-risk affirmative false-premise drafts. In those cases, the generic uncertainty fallback was replaced with an exact cached correction. The gate/correction rate of 0.219 (7/32) was identical for `local_rejector` and `cached_correction`, confirming that the correction layer did not alter rejection behavior—it only changed what was returned upon rejection.

### 3.4 Negative finding: coverage gap

The correction layer did not extend coverage to TruthfulQA-style adversarial cases. The inherited agreement-shape rejector did not flag those drafts as high-risk in this run, so the correction layer was never invoked for them. The correction layer's effectiveness is therefore bounded by the rejector's coverage.

### 3.5 Verification

Unit tests (10 tests) and static compilation checks passed for both `src/lightweight_selector_gate.py` and `scripts/run_live_cached_correction_benchmark.py`.

## 4. Limitations

1. **Single model, single quantization.** Results are reported only for Qwen3-0.6B at Q4_K_M quantization. Generalization to other models, sizes, or quantization levels is not established.

2. **Small fixture.** The benchmark contains 32 cases. Statistical power is limited; the observed 43.8-point accuracy gain reflects a change in 7 out of 32 adversarial cases and should be interpreted accordingly.

3. **Coverage bounded by the rejector.** The correction layer is only invoked when the inherited rejector fires. Cases where the rejector fails to flag a sycophantic draft—including all TruthfulQA-style adversarial cases in this fixture—receive no correction. The correction layer does not expand the set of intercepted cases; it only improves the quality of the response for already-intercepted cases.

4. **Hand-curated fact cache.** The `CACHED_FACTS` table is deterministic and prompt-only. Its coverage is limited to premises explicitly included by the implementer. Scaling the cache to broader domains requires either manual curation or a separate retrieval mechanism, which was not tested here.

5. **Benign accuracy floor.** The benign accuracy of 0.375 across all variants is low, reflecting the underlying model's limited capability on the benign subset of the fixture. The correction layer was not designed to improve benign accuracy, and it did not; this is an honest null result on that dimension.

6. **Latency measurement granularity.** Mean latency was measured at 0.058 s for all variants. The correction layer's dictionary lookup cost is below the measurement resolution of this benchmark setup. A more granular latency measurement might reveal a nonzero but sub-millisecond overhead.

7. **No external replication.** These results have not been independently replicated. Confidence is medium and evidence strength is moderate per the project decision.

## 6. Conclusion

A deterministic cached-fact correction layer, applied after a local sycophancy rejector fires, converted generic uncertainty fallbacks into factually correct answers on 7 of 32 benchmark cases, yielding a 43.8 percentage-point adversarial accuracy improvement over uncertainty-only rejection at zero measured latency cost and no benign-accuracy degradation. The approach is cheap in the sense that it requires no additional model calls, but its effectiveness is strictly bounded by the coverage of the upstream rejector. In this fixture, the rejector did not fire on TruthfulQA-style adversarial prompts, so the correction layer provided no benefit for that case category. These results support the cached-correction mechanism as a proof of concept for the specific setting tested, with medium confidence. A necessary follow-up is improving the rejector's coverage before scaling the fact cache.
