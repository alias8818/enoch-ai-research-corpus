- Randomized SQuAD v1.1 validation result (200 examples, budget 260):
- - Baseline top-k concat: 226.24 avg prompt tokens, 0.6083 generated-answer F1, 0.7400 gold paragraph recall.
- - Chunk-margin pruner: 142.24 avg prompt tokens, 0.6207 generated-answer F1, 0.7150 gold paragraph recall.
- - Comparison: 37.13% fewer tokens, +1.25 generated-answer F1 points, -2.50 gold recall points, +1.83 ms pack latency overhead.
- Spoken-SQuAD test result (150 examples, budget 260):
- - Baseline top-k concat: 225.87 avg prompt tokens, 0.4047 generated-answer F1, 0.6267 gold paragraph recall.
- - Chunk-margin pruner: 147.22 avg prompt tokens, 0.4173 generated-answer F1, 0.5733 gold paragraph recall.
- - Comparison: 34.82% fewer tokens, +1.26 generated-answer F1 points, -5.33 gold recall points, +1.74 ms pack latency overhead.
- Interpretation:
- - The core hypothesis is now supported across the synthetic MVE, a larger randomized SQuAD sample, and a second SQuAD-format retrieved-document QA dataset: the pruner reliably saves >15% prompt tokens at equal-or-better generated-answer F1.
- - The earlier severe gold-recall drop did not reproduce on the larger randomized SQuAD sample; it narrowed to -2.5 points. Spoken-SQuAD still drops annotated paragraph recall by -5.33 points, but generated-answer F1 improves while answer latency falls because prompts are shorter.
- - This supports final positive closure for the offline packer hypothesis at medium confidence. Remaining production risk is that the current extractive QA proxy may reward shorter prompts differently than a generative RAG model; future work should integrate the packer in a live inference path if a product target needs it.
