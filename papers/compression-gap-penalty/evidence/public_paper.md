## Abstract

Lossy context compression for large language models risks discarding answer-critical information. We propose the Compression Gap Penalty (CGP), a lightweight text-only metric that quantifies semantic atom loss between a full context and its compressed form. CGP computes, for each fact atom in the original context, the maximum TF-IDF cosine similarity to any sentence in the compressed context, then reports the weighted mean unrecovered atom mass. We evaluate three weighting schemes—uniform, query-overlap, and focused (squared) query-overlap—on 1,000 synthetic deterministic QA tasks across five compressors and five compression budgets (25,000 total evaluation rows). The focused query CGP achieves ROC-AUC of 0.736 for predicting downstream failure, with a high-precision operating point (95.6% precision at 21.3% recall at the top-decile threshold). However, the metric is moderate in absolute discriminative power and exhibits an important failure mode: task-correct compressors that retain only the answer atom can still register high global CGP. Results are limited to synthetic lexical tasks with deterministic correctness criteria. We conclude that CGP is viable as a conservative high-risk flag in controlled settings but is insufficient as a standalone production gate without validation on real LLM benchmarks.

## 1. Introduction

Context compression—reducing the token count of a prompt before inference—is an increasingly common technique for managing long-context costs and latency. However, lossy compression may remove information required to answer a given query, and the compression ratio alone does not predict whether critical content survives.

We investigate whether a cheap, text-only metric computed after compression can flag cases where answer-critical information has been lost, without requiring a forward pass through the target model. The core idea is to decompose the original context into semantic atoms (fact-bearing sentences), measure how well each atom is recovered in the compressed output via TF-IDF cosine similarity, and aggregate the unrecovered mass into a single penalty score.

This paper reports results from a controlled synthetic evaluation. We do not claim production readiness; rather, we document the discriminative performance, threshold behavior, and failure modes of the approach as a research prototype, and specify the additional evidence required before deployment.

## 2. Method

### 2.1 Compression Gap Penalty Definition

Let the original context $C$ contain a set of semantic atoms $A = \{a_1, \ldots, a_n\}$, where each atom is a fact-bearing sentence. Let $S = \{s_1, \ldots, s_m\}$ be the sentences in the compressed context $C'$. For each atom $a_i$, define the recovery score:

$$\text{recovery}(a_i) = \max_{s \in S} \cos_{\text{tfidf}}(a_i, s)$$

The atom gap is:

$$\text{gap}(a_i) = 1 - \text{recovery}(a_i)$$

CGP is the weighted mean of atom gaps:

$$\text{CGP} = \frac{\sum_{i} w_i \cdot \text{gap}(a_i)}{\sum_{i} w_i}$$

We evaluate three weighting schemes:

- **Uniform CGP:** $w_i = 1$ for all atoms.
- **Query CGP:** $w_i = |a_i \cap Q|$, the token overlap between atom $a_i$ and the question $Q$.
- **Focused query CGP:** $w_i = |a_i \cap Q|^2$, squaring the overlap to emphasize atoms that share multiple question tokens.

Additionally, we compute an **answer atom gap** as an oracle diagnostic: the gap for the known answer-bearing atom. This is not deployable (it requires knowing the answer) but validates the mechanism's ceiling.

### 2.2 Synthetic Task Design

Each task instance contains:

- 60 generated fact atoms, each stating a code value for a named entity (e.g., "Fact K004: the orbit code is garnet-6796. This note mentions beacon and ledger.").
- 60 distractor background lines with no answer-bearing content.
- A question targeting one specific fact (e.g., "What is the orbit code for K004?").
- A deterministic correctness criterion: the compressed context is correct if and only if it retains the answer sentence (or its value).

This is a toy simulation: the tasks are single-hop, the facts are formulaic, and correctness is deterministic rather than assessed by an LLM.

### 2.3 Compressors

Five compressors are evaluated, each receiving a token budget expressed as a fraction of the full context length:

| Compressor | Strategy |
|---|---|
| `head` | Retain the first tokens up to budget |
| `tail` | Retain the last tokens up to budget |
| `random` | Retain a random subset of sentences up to budget |
| `query_overlap` | Rank sentences by token overlap with the question, retain top-scoring up to budget |
| `oracle_answer_first` | Place the answer sentence first, then fill remaining budget with other sentences |

### 2.4 Compression Budgets

Five token-budget ratios are tested: 0.05, 0.1, 0.2, 0.3, and 0.5 of the full-token count.

### 2.5 Evaluation Protocol

The full evaluation generates 1,000 task instances × 5 compressors × 5 ratios = 25,000 evaluation rows. For each row, we compute all CGP variants and record downstream correctness. Predictive performance is measured via ROC-AUC and average precision for the failure class, plus threshold-level precision/recall analysis.

A preliminary smoke test was conducted with 5 tasks, 12 facts, 12 distractors, and ratios 0.2 and 0.5 (50 rows) to validate the harness before the full run. The smoke test used seed 0; the full run seeds are determined by task index (deterministic).

## 3. Results

### 3.1 Predicting Downstream Failure

All three deployable CGP variants are positively associated with downstream failure. Focused query weighting consistently outperforms uniform and linear query weighting:

| Metric | ROC-AUC (failure) | Avg. precision (failure) | Mean when correct | Mean when failed | Δ (failed − correct) |
|---|---:|---:|---:|---:|---:|
| Focused query CGP | 0.7361 | 0.7197 | 0.4615 | 0.6021 | 0.1406 |
| Query CGP | 0.7040 | 0.6774 | 0.4793 | 0.6000 | 0.1207 |
| Uniform CGP | 0.6816 | 0.6476 | 0.4888 | 0.5990 | 0.1102 |
| Answer atom gap (oracle) | 1.0000 | 1.0000 | ≈0 | 0.7396 | 0.7396 |

The oracle metric achieves perfect discrimination, confirming that atom-level gap is a valid signal when the answer-bearing atom is known. The deployable variants capture a fraction of this signal, with focused query CGP recovering the most. The gap between the best deployable metric (0.736) and the oracle ceiling (1.0) is substantial, indicating room for improvement in atom identification and weighting.

### 3.2 Threshold Behavior

Focused query CGP supports a conservative high-precision operating regime:

| Flagged percentile | Threshold | Precision | Recall | False-positive rate |
|---:|---:|---:|---:|---:|
| Top 50% | 0.5506 | 0.606 | 0.675 | 0.358 |
| Top 40% | 0.5861 | 0.633 | 0.564 | 0.266 |
| Top 30% | 0.6553 | 0.698 | 0.466 | 0.165 |
| Top 20% | 0.6867 | 0.756 | 0.337 | 0.089 |
| Top 10% | 0.7352 | 0.956 | 0.213 | 0.008 |

At the top-decile threshold, 95.6% of flagged instances are genuine failures, but 78.7% of failures go undetected. CGP is thus best interpreted as a high-risk flag rather than a comprehensive failure detector. The precision–recall tradeoff is steep: moving from the top-decile to the top-quartile threshold more than doubles recall (21.3% → 46.6%) but drops precision from 95.6% to 69.8%.

### 3.3 Compressor-Level Analysis

Mean performance across all budgets:

| Compressor | Accuracy | Mean focused CGP | Mean answer atom gap |
|---|---:|---:|---:|
| `query_overlap` | 1.0000 | 0.3715 | ≈0 |
| `oracle_answer_first` | 1.0000 | 0.5441 | ≈0 |
| `head` | 0.2818 | 0.5682 | 0.5312 |
| `tail` | 0.2534 | 0.5680 | 0.5516 |
| `random` | 0.2196 | 0.5714 | 0.5778 |

A critical observation: `oracle_answer_first` achieves perfect task accuracy while registering high focused CGP (0.5441). This compressor retains the answer atom but discards most other atoms, yielding high global information loss despite task correctness. This demonstrates that CGP measures information loss/risk, not task accuracy per se. A compressor that is narrowly correct but globally lossy will still be flagged. Whether this is desirable depends on the application: for single-hop code extraction it is a false alarm, but for tasks requiring broader context it may be a genuine risk signal.

The three lossy compressors (`head`, `tail`, `random`) cluster closely on focused CGP (0.568–0.571) despite modest differences in accuracy (0.22–0.28), suggesting that CGP is capturing a shared property of aggressive truncation rather than discriminating finely among lossy strategies.

### 3.4 Smoke Test Validation

The preliminary smoke test (50 rows, 5 tasks, ratios 0.2 and 0.5) confirmed the harness was functioning correctly. Smoke-test correlations with failure were directionally consistent with the full run: query CGP point-biserial $r = 0.465$ ($p < 0.001$), uniform CGP $r = 0.404$ ($p = 0.004$), and oracle answer atom gap $r = 0.995$ ($p \approx 10^{-51}$). The focused query CGP variant was not yet implemented at smoke-test time.

## 4. Limitations

1. **Synthetic lexical tasks.** Generated fact atoms use formulaic sentence structures with named codes, favoring exact and near-exact lexical matching. Real tasks involve paraphrase, implicit dependencies, multi-hop reasoning, and indirect references that TF-IDF cosine similarity will under-model.

2. **Deterministic correctness criterion.** Correctness is defined as retention of the answer sentence in the compressed context. Real LLM behavior may tolerate partial information, draw inferences from remaining context, or fail even when the answer atom is nominally present.

3. **TF-IDF as recovery proxy.** TF-IDF cosine similarity captures lexical overlap but not semantic equivalence. Embedding-based recovery or model-logprob gap measures may perform differently and are not evaluated here.

4. **Moderate discriminative power.** The best deployable variant (focused query CGP) achieves ROC-AUC of 0.736. This is above chance but well below the oracle ceiling. At high-precision operating points, recall is low (21.3% at top decile). CGP alone is insufficient as a standalone accept/reject gate.

5. **Global loss vs. task correctness ambiguity.** As demonstrated by the `oracle_answer_first` compressor, high CGP does not necessarily imply task failure. The metric conflates global information loss with task-critical information loss. A task-focused variant (e.g., one that identifies the most likely answer-bearing atoms) would be needed for higher precision, but this requires information that may not be available at compression time.

6. **No real-model validation.** All results are from synthetic tasks with deterministic evaluation. No LLM was used to score downstream correctness. The relationship between CGP and real model performance remains unestablished.

7. **Incomplete dependency recording.** Specific Python package versions were not recorded in the run notes, which is a gap in the reproducibility record.

## 6. Conclusion

The Compression Gap Penalty is a lightweight, text-only metric that can flag instances where lossy context compression has discarded information. In a controlled synthetic evaluation (1,000 tasks, 25,000 evaluation rows), the focused query CGP variant achieves ROC-AUC of 0.736 for predicting downstream failure, with a usable high-precision operating point (95.6% precision at 21.3% recall). Query-focused weighting improves over uniform weighting, confirming that directing the penalty toward query-relevant atoms is beneficial.

However, the metric has significant limitations. It is moderate in absolute discriminative power, conflates global information loss with task-critical loss (producing false alarms for narrowly correct but globally lossy compressors), and has been evaluated only on synthetic lexical tasks with deterministic correctness criteria. The gap between the deployable metric (AUC 0.736) and the oracle ceiling (AUC 1.0) is large, indicating substantial room for improvement—likely through better atom identification, embedding-based recovery, or model-informed gap measures.

We recommend proceeding to a real-model evaluation phase: replacing synthetic tasks with 100–300 real QA or code tasks, scoring correctness with an actual LLM, and comparing TF-IDF CGP against embedding CGP, answer/span-aware CGP, and model-logprob gap. CGP should be adopted only if it improves failure prediction over compression ratio alone by at least +0.10 ROC-AUC and provides a useful high-precision risk threshold on real tasks.
