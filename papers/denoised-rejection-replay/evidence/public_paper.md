## Abstract

Speculative decoding accelerates autoregressive inference by proposing tokens from a small draft model and verifying them against a target model via rejection sampling. Rejected proposals are typically discarded or used as per-token negative examples for draft adaptation. We derive an alternative: under the standard single-token speculative acceptance rule, the difference between residual replacement token counts and rejected proposal counts yields an unbiased estimate of the signed draft–target distribution error, $\mathbb{E}[\text{replacement}_i - \text{rejection}_i] / \text{visits} = p_i - q_i$. Aggregating this signal over a replay window and applying it as a multiplicative update to the draft distribution defines Denoised Rejection Replay (DRR). In a synthetic categorical simulation (256 contexts, 128-token vocabulary, 80 replay rounds, 8 seeds), DRR achieved a mean acceptance rate of 0.876 versus 0.788 for naive per-event replay and 0.620 for a static draft, recovering approximately 67% of the acceptance gap between the static draft and an oracle target blend. DRR's advantage increased with replay-window sample size and was present but modest under sparse traffic (500 requests/round: 0.678 vs. 0.637 naive). These results establish algorithmic viability in a toy simulation; real-model validation on an actual speculative decoding serving stack remains necessary before any production claim.

## Introduction

Speculative decoding generates multiple candidate tokens from a lightweight draft model and verifies them in parallel against a larger target model, preserving the target distribution through a rejection-sampling rule. The method's speedup depends directly on the acceptance rate: the fraction of draft tokens the target model endorses. A draft model closely matching the target distribution yields high acceptance and large speedups; a poor match wastes computation on rejected proposals.

When a draft token is rejected, the standard algorithm samples a replacement from the residual distribution $r \propto \max(p - q, 0)$, ensuring the output distribution equals $p$. These rejection events carry information about where $q$ diverges from $p$. A natural idea is to use this information to improve the draft model. Naive approaches treat each rejection as a negative example and each acceptance as a positive example, applying sparse per-event updates. However, individual events are noisy realizations of the underlying distribution gap, and per-token nudges can be high-variance, particularly for low-traffic contexts.

This work asks: can rejected speculative-decoding proposals be replayed as a *denoised* learning signal, rather than as noisy per-token negative examples? We show that under the standard acceptance rule, the aggregate difference between replacement token counts and rejected proposal counts directly estimates $p - q$ without additional target-model invocations. Aggregating this signed residual over a replay window before updating the draft yields a lower-variance adaptation signal. We call this method Denoised Rejection Replay (DRR).

We evaluate DRR in a synthetic categorical simulation and compare it against a static (unadapted) draft, naive per-event replay, and an oracle that blends directly toward the target. We also ablate the effect of replay-window sample size on adaptation quality.

## Method

### Speculative Decoding Setup

Consider a context with target distribution $p$ over a vocabulary and draft distribution $q$. The draft proposes a token $y \sim q$. The verifier accepts $y$ with probability $\min(1, p[y]/q[y])$. If rejected, a replacement token $z$ is sampled from the residual distribution $r \propto \max(p - q, 0)$.

### Signed Residual Identity

Under this acceptance rule, two conservation properties hold:

1. The probability that the rejected token equals $i$ is $\max(q_i - p_i, 0)$ — the excess mass of $q$ over $p$ at token $i$.
2. The probability that the residual replacement token equals $i$ is $\max(p_i - q_i, 0)$ — the deficit mass of $q$ relative to $p$ at token $i$.

Therefore, for each token $i$:

$$\frac{\mathbb{E}[\text{replacement\_count}_i - \text{rejection\_count}_i]}{\text{visits}} = p_i - q_i$$

This identity requires no additional target-model evaluations beyond those already performed during speculative decoding. The signal is available in standard speculative decoding traces at no extra inference cost.

### Denoised Rejection Replay Update

DRR aggregates the signed residual counts over a replay window of $N$ requests, then updates the draft distribution:

$$q \leftarrow \text{normalize}(\max(q + \eta \cdot \Delta, \epsilon))$$

where $\Delta_i = \text{replacement\_count}_i - \text{rejection\_count}_i$ over the window, $\eta$ is a step size, and $\epsilon$ is a small floor to maintain positivity. This contrasts with naive event replay, which applies per-event positive/negative nudges for individual accepted/rejected tokens as they occur, without aggregation.

### Simulation Design

We implemented a synthetic categorical simulator with the following configuration:

- **Contexts:** 256, each with an independent target distribution
- **Vocabulary:** 128 tokens
- **Target distributions:** Moderately spiky (non-uniform, concentrated)
- **Draft initialization:** Perturbed version of the target (noisy, stale)
- **Traffic:** Non-uniform across contexts
- **Speculative block length:** 6 (for speedup proxy computation)
- **Draft cost proxy:** 0.12 verifier-equivalent per token
- **Step size $\eta$:** 0.1

We compared four methods:

1. **Static:** No draft adaptation
2. **Naive event replay:** Per-event positive/negative updates for accepted/rejected tokens
3. **DRR:** Aggregated signed residual update as described above
4. **Oracle target blend:** Direct blending toward the target distribution (upper bound)

The speedup proxy is computed as:

$$\text{speedup} = \frac{\text{block\_length}}{1 + (\text{block\_length} - 1) \cdot \text{acceptance} + \text{draft\_cost} \cdot \text{block\_length}}$$

This proxy captures the first-order relationship between acceptance rate and speculative decoding speedup but omits kernel efficiency, KV-cache behavior, batching, memory bandwidth, and scheduling effects present in real serving stacks.

## Results

### Main Experiment

Configuration: 80 replay rounds, 8,000 served requests per round, 8 random seeds.

| Method | Acceptance (mean) | Acceptance (p10) | L1(q,p) (mean) | Speedup proxy | Acceptance gain vs static |
|---|---:|---:|---:|---:|---:|
| Static | 0.6198 | 0.5525 | 0.7604 | 1.4920 | 0.0000 |
| Naive event replay | 0.7882 | 0.6478 | 0.4236 | 2.3311 | +0.1684 |
| DRR | 0.8762 | 0.6624 | 0.2477 | 3.0077 | +0.2564 |
| Oracle target blend | 0.9999 | 0.9999 | 0.0002 | 4.0688 | +0.3801 |

DRR recovered approximately 67% of the available acceptance gap between the static draft and the oracle: $(0.8762 - 0.6198) / (0.9999 - 0.6198) \approx 0.674$.

The L1 distance between draft and target distributions decreased from 0.7604 (static) to 0.2477 (DRR), compared to 0.4236 for naive event replay and near-zero for the oracle.

### Adaptation Trajectory

| Round | Static accept | Naive accept | DRR accept | Oracle accept |
|---:|---:|---:|---:|---:|
| 0 | 0.620 | 0.620 | 0.620 | 0.620 |
| 10 | 0.620 | 0.673 | 0.793 | 0.867 |
| 20 | 0.620 | 0.703 | 0.849 | 0.954 |
| 40 | 0.620 | 0.743 | 0.874 | 0.994 |
| 80 | 0.620 | 0.788 | 0.876 | 1.000 |

All methods start from the same initial draft. DRR converges faster than naive event replay in the early rounds (round 10: 0.793 vs. 0.673) and reaches a higher asymptote. Both adaptive methods plateau by approximately round 40–60, with DRR settling near 0.876 and naive near 0.788. DRR does not reach oracle performance, leaving a residual gap of approximately 0.124 in acceptance rate.

### Request-Volume Ablation

After 40 rounds, 5 seeds each:

| Requests/round | Static accept | Naive accept | DRR accept | DRR p10 |
|---:|---:|---:|---:|---:|
| 500 | 0.620 | 0.637 | 0.678 | 0.577 |
| 1,000 | 0.620 | 0.651 | 0.725 | 0.595 |
| 2,000 | 0.620 | 0.672 | 0.777 | 0.613 |
| 8,000 | 0.620 | 0.742 | 0.875 | 0.675 |

DRR's advantage over naive event replay is present at all traffic levels but grows substantially with replay-window sample size. At 500 requests per round, DRR acceptance is 0.678 versus 0.637 for naive — a modest gain of 0.041. At 8,000 requests per round, the gap widens to 0.875 versus 0.742, a gain of 0.133. This is consistent with the premise that DRR denoises a signed residual estimate; aggregation requires sufficient samples per context to reduce variance. DRR is not a single-event learner, and its benefit scales with the statistical reliability of the aggregated signal.

The p10 (10th percentile across seeds) for DRR at 500 requests/round is 0.577, below the static mean of 0.620, indicating that under sparse traffic and limited seeds, some runs produce noisy updates that can temporarily degrade the draft. This variability shrinks at higher traffic volumes (p10 at 8,000 requests/round: 0.675).

### Resource Usage

The main simulation run completed on a single CPU in approximately 20 seconds with a maximum RSS of 58,416 KB. This was a synthetic categorical simulation, not a GPU benchmark. No CUDA, kernel-level, or hardware-specific calibration was involved.

## Limitations

1. **Synthetic categorical simulation, not a real LLM.** The simulator assumes categorical distributions are directly updateable via multiplicative adjustment. Real draft models are neural networks; applying the signed residual signal would require a parameter update (e.g., via a lightweight adapter or head fine-tuning), and the mapping from distributional targets to weight gradients introduces additional approximation. The simulation results likely overstate the gains achievable with neural draft models, where gradient noise, capacity constraints, and optimization dynamics may limit how closely the model can track the target distribution.

2. **Stationary target distributions.** The simulation assumes target distributions do not shift within a replay window. In production serving, context distributions drift over time as user requests change. DRR's aggregation window must be short enough to track non-stationarity but long enough to accumulate reliable estimates — a tension not explored here.

3. **Speedup proxy, not measured throughput.** The speedup proxy assumes a simple model of speculative decoding speedup based on acceptance rate, block length, and draft cost ratio. Real throughput depends on kernel efficiency, KV-cache behavior, batching, memory bandwidth, and scheduling — none of which are captured by the proxy. The proxy likely overstates the speedup from acceptance gains alone.

4. **No domain shift evaluation.** All contexts in the simulation share the same vocabulary and distributional family. Production speculative decoding serves diverse domains where the draft–target gap varies substantially. DRR's behavior under distribution shift between training and serving contexts is untested.

5. **Sparse-traffic risk.** The p10 results at 500 requests/round show that DRR can degrade the draft when the aggregated signal is noisy. Conservative step sizes or minimum-sample thresholds per context may be necessary in low-traffic regimes, but these mitigations were not evaluated.

6. **No comparison to other draft-adaptation methods.** The comparison is limited to static draft and naive per-event replay. Other approaches — such as distillation from the target, online KL-matching, or reward-model-guided adaptation — were not evaluated and may outperform DRR.

7. **Residual gap not explained.** DRR recovers approximately 67% of the acceptance gap, leaving 33% unrecovered. The simulation does not isolate whether this residual is due to step-size limitations, the multiplicative update form, the positivity floor, or fundamental properties of the aggregated estimator. Understanding and closing this gap is an open question.

## Conclusion

We have shown that under the standard speculative decoding rejection rule, the difference between residual replacement counts and rejected proposal counts provides an unbiased estimate of $p - q$ at no additional inference cost. Aggregating this signal over a replay window and applying it as a draft distribution update — Denoised Rejection Replay — substantially outperformed naive per-event replay in a synthetic categorical simulation, recovering approximately 67% of the acceptance gap between a static draft and an oracle target blend.

The method's advantage is real but bounded: DRR does not reach oracle performance, and its benefit depends on sufficient traffic per context to denoise the signed residual estimate. Under sparse traffic (500 requests/round), the advantage is modest and variance is high, with some seeds producing draft degradation.

This work establishes the algorithmic identity and synthetic viability of DRR. It does not establish that DRR works on real language models or real serving stacks. The critical next step is validation on an actual speculative decoding pair or collected target/draft traces, measuring acceptance length and wall-clock throughput under domain shift. The signed residual identity holds regardless of the model class; whether neural draft models can effectively absorb this signal through parameter updates remains an open empirical question.
