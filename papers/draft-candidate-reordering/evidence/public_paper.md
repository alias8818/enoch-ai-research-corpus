## Abstract

Speculative decoding accelerates autoregressive language model inference by proposing draft tokens that a target model verifies in a single forward pass. This study investigates whether reordering draft candidates before verification can improve acceptance rates. We test two mechanistically distinct interpretations: (1) intra-chain token reordering, where tokens within a single autoregressive draft sequence are permuted before verification, and (2) independent candidate scheduling, where multiple internally-ordered candidate branches are reordered before a fixed-budget verifier. Using a deterministic synthetic Markov language model harness (vocabulary size 8, sequence length 3), we find that naive intra-chain reordering produces a large distribution shift (total variation distance 0.513 against the target distribution, versus 0.015 for exact speculative sampling), rendering it unviable without a corrected proposal distribution. In contrast, score-sorted scheduling of independent candidates under a fixed verification budget of 4 out of 12 candidates yields a 62.8% increase in mean accepted tokens over input-order scheduling (4.87 vs. 2.99), with a 62.4% win rate, though 37.6% of the oracle gap remains. These results are limited to synthetic Markov-model conditions and do not constitute validation on production language models.

## 1. Introduction

Speculative decoding reduces inference latency by using a fast draft model to propose candidate token sequences, which a larger target model then verifies in a single forward pass. A rejected token invalidates all subsequent tokens in the draft chain because autoregressive generation conditions each token on its predecessors. This dependency structure raises a natural question: can reordering draft candidates before verification improve the expected number of accepted tokens?

The phrase "draft candidate reordering" is ambiguous. It may refer to at least two mechanistically distinct operations:

1. **Intra-chain token reordering**: Permuting tokens within a single sampled autoregressive draft sequence before presenting them to the verifier. The appeal is that apparently high-probability tokens might be moved earlier to survive rejection cascades.
2. **Independent candidate scheduling**: Reordering multiple complete, internally-ordered candidate branches before a verifier that can evaluate only a fixed number of them. The appeal is that better-calibrated candidates receive verification budget before worse ones.

These two interpretations have fundamentally different implications for the correctness of the sampling distribution. Intra-chain reordering alters the conditional dependency structure that speculative decoding's acceptance criterion relies upon. Independent candidate scheduling preserves each branch's internal token order and proposal probability accounting, changing only the allocation order of a scarce verification resource.

This paper reports experiments evaluating both interpretations under controlled synthetic conditions and documents the constraints under which each is or is not viable.

## 2. Method

### 2.1 Synthetic Markov Language Model

All experiments use a deterministic synthetic Markov language model implemented in `scripts/draft_candidate_reordering_eval.py`. The model has:

- **Vocabulary size**: 8 tokens
- **Sequence length scored**: 3 tokens
- **Draft block size**: 4 tokens

The Markov model generates token probabilities via a fixed transition matrix, providing a controlled environment where the true target distribution is known exactly. This permits computation of total variation distance (TVD) between sampling strategies and the ground-truth target distribution, which is not feasible with real language models.

### 2.2 Experiment 1: Intra-Chain Token Reordering

We compare three sampling strategies:

1. **Target sampling**: Direct sampling from the true target distribution (ground truth).
2. **Exact speculative sampling**: Standard speculative decoding with correct acceptance ratios. This is the baseline correct implementation.
3. **Naive reordered sampling**: Sample an autoregressive draft chain, then sort the draft tokens by their individual proposal probabilities (highest first) before applying standard speculative verification acceptance criteria.

The metric is total variation distance (TVD) from the target distribution. If naive reordering preserves the sampling distribution, TVD should be comparable to exact speculative sampling. If it introduces distribution shift, TVD will be substantially larger.

We ran 300,000 trials for this experiment.

### 2.3 Experiment 2: Independent Candidate Scheduling

We model a setting where a pool of candidate sequences is available and the verifier can evaluate only a fixed number:

- **Candidate pool size**: 12 independent branches
- **Verification budget**: 4 candidates evaluated per step
- Each candidate branch is internally autoregressively ordered; only the scheduling order among branches varies.

Three scheduling policies are compared:

1. **Input-order**: Candidates are verified in the order they were generated.
2. **Score-sorted**: Candidates are ranked by a calibration score (derived from draft-model probabilities) and verified best-first.
3. **Oracle**: Candidates are ranked by their true acceptance outcome (unimplementable in practice, providing an upper bound).

The primary metric is mean accepted tokens per verification step. We also report the win rate of score-sorted over input-order scheduling and the remaining gap to the oracle.

We ran 200,000 scheduler batches for this experiment.

### 2.4 Execution Environment

All experiments executed on host GB10 (`Linux gx10-efe8`, aarch64) with swap intentionally disabled. The synthetic harness is CPU-only; the GPU remained at 0% utilization and approximately 11 W idle power throughout. Maximum RSS was 15,568 kB. ## 3. Results

### 3.1 Experiment 1: Intra-Chain Reordering Distribution Shift

| Sampling Strategy | TVD vs. Target |
|---|---|
| Exact speculative | 0.01469 |
| Naive reordered | 0.51286 |

The exact speculative sampler's TVD of 0.01469 is consistent with Monte Carlo sampling noise around the true distribution. The naive reordered sampler's TVD of 0.51286 represents a severe distribution shift — approximately 35× the exact sampler's TVD.

This is a **negative result**: naive intra-chain token reordering, where draft tokens are sorted by apparent probability and then verified using standard acceptance ratios, does not preserve the target sampling distribution. The acceptance criterion in speculative decoding is derived under the assumption that draft tokens appear in their autoregressive sampling order. Permuting that order invalidates the acceptance ratios without a corresponding correction to the proposal distribution.

### 3.2 Experiment 2: Independent Candidate Scheduling

| Scheduling Policy | Mean Accepted Tokens |
|---|---|
| Input-order | 2.9883 |
| Score-sorted | 4.8651 |
| Oracle | 7.7907 |

| Metric | Value |
|---|---|
| Score-sorted lift vs. input-order | +62.80% |
| Score-sorted win rate vs. input-order | 62.35% |
| Remaining gap to oracle | 37.55% |

Score-sorted scheduling substantially outperforms input-order scheduling, increasing mean accepted tokens by 62.8%. The 62.4% win rate indicates that score-sorted scheduling wins in a majority of head-to-head batch comparisons, though it does not dominate input-order in every instance.

However, score-sorted scheduling captures only 62.4% of the theoretical maximum improvement (the oracle gap is 37.6%), indicating that the draft-model calibration score is an imperfect predictor of verification outcome. The remaining gap reflects the fundamental information asymmetry between draft-model scores and target-model acceptance decisions.

### 3.3 Calibration Run Consistency

A calibration run with 50,000 trials and 50,000 scheduler batches (Experiment 1: length 3; Experiment 2: same parameters) produced consistent directional results with the final run, confirming that the effects are not artifacts of small sample sizes. A smoke run with 2,000 trials and 5,000 scheduler batches also showed the same directional pattern.

## 4. Limitations

1. **Synthetic model only.** All results derive from a synthetic Markov language model with vocabulary size 8 and sequence length 3. Real language models have vocabularies of 30,000–250,000 tokens, much longer sequences, and far more complex dependency structures. The magnitude of both the distribution shift (Experiment 1) and the scheduling lift (Experiment 2) may differ substantially in production settings.

2. **No real-model validation.** No experiments were conducted with actual draft/target model pairs (e.g., llama.cpp hook prototypes, CUDA copy calibrations, or production inference pipelines). The results constitute toy simulation evidence, not final production validation.

3. **Calibration score quality.** The 37.6% remaining oracle gap in Experiment 2 depends on the quality of the calibration score used for scheduling. With real draft models, calibration quality may be better or worse than in this synthetic setting. The lift percentage is an upper bound on what a comparably-calibrated real system would achieve, but not a guaranteed lower bound.

4. **Fixed budget and pool size.** Experiment 2 uses a single configuration (pool=12, budget=4). The sensitivity of the scheduling lift to these parameters was not characterized.

5. **Intra-chain reordering not exhaustively ruled out.** The negative result applies specifically to *naive* intra-chain reordering with unmodified acceptance ratios. A corrected approach that derives a new exact proposal distribution and corresponding acceptance criterion for permuted draft sequences is not ruled out in principle, but would require substantial theoretical work and is not evaluated here.

6. **No latency measurement.** The experiments measure accepted-token counts, not wall-clock latency. The overhead of computing calibration scores and sorting candidates may offset some of the acceptance-rate gains in practice.

7. **Claim audit status.** The claim ledger for this artifact was flagged as `blocked_empty_claims` at the time of draft generation, meaning no structured claims were extracted for formal evidence audit. The results reported here should be treated accordingly.

## 6. Conclusion

Draft candidate reordering is not a monolithic technique; its viability depends critically on what is reordered and what verification invariants are preserved.

**Intra-chain token reordering is not viable in its naive form.** Sorting tokens within an autoregressive draft sequence and applying standard speculative decoding acceptance ratios produces a total variation distance of 0.513 from the target distribution — a severe distributional corruption. This result is consistent with the theoretical structure of speculative decoding, where acceptance probabilities are derived under the assumption of autoregressive token ordering. Any implementation of intra-chain reordering would require deriving a corrected proposal distribution and acceptance criterion, which is left to future work.

**Independent candidate scheduling is viable and beneficial under constrained conditions.** When each candidate branch preserves its internal token order and proposal probability accounting, reordering branches by a calibration score before a fixed-budget verifier yields a 62.8% increase in mean accepted tokens over input-order scheduling. This improvement is substantial but incomplete, with 37.6% of the oracle gap remaining due to imperfect calibration.

The recommended implementation path is a branch or candidate scheduler for tree-structured or multi-candidate speculative decoding, with the invariant that each branch's token order and proposal-probability accounting remain intact. Naive token sorting inside a sampled speculative chain should be explicitly avoided.
