## Research question

Can a lightweight ledger of falsifiable intermediate claims reduce unchecked false claims and improve task accuracy for a small reasoning agent?

## Concrete hypothesis

On synthetic multi-hop tabular reasoning tasks with noisy agent memory, a ledger that requires each intermediate fact and calculation to be checked against a verifier will reduce accepted false claims and improve final-answer accuracy versus the same agent without the ledger.

## Experiment design

- Directly tested: a deterministic small-agent harness with three conditions:
  - `baseline`: uses noisy memory and no claim ledger.
  - `ledger_facts`: verifies and repairs fact claims before use.
  - `ledger_full`: verifies and repairs fact claims plus arithmetic/selection claims.
- Tasks: generated tabular worlds over groups and attributes, with `sum_two`, `difference`, `max_group`, and `count_above` queries.
- Noise model: each remembered fact is independently corrupted with probability 0.25 by a bounded integer perturbation.
- Metrics: final-answer accuracy, checked claims, rejected claims, accepted false claims, and latency.

## Resource calibration

This is CPU-only, single-process Python and is intentionally scoped below 15 minutes. Expected wall clock for the main run is under 10 seconds on the 8-vCPU CPU worker, memory under 100 MB, no GPU use, and no checkpoints required because the run is short. GB10 is not required or used.

## Interpretation

The falsifiable ledger mechanism is supported in the controlled setting. When the small agent's memory is corrupted, the baseline accepts false intermediate claims and final accuracy falls. The ledger conditions expose and repair those claims before they affect the answer, eliminating accepted false claims and restoring accuracy on these tasks. The no-noise control shows the ledger does not create an artificial benefit when there are no false claims to catch.

The result is useful but not paper-ready. The verifier is a perfect synthetic oracle over generated tables, and the "small reasoning agent" is a deterministic harness rather than a learned LLM or deployed tool-using agent. This directly tests the claim-ledger mechanism under controlled corruption, not whether real small language models will reliably formulate verifiable claims, call verifiers, or preserve answer quality under realistic evidence retrieval failures.

## Limitations

- Synthetic generated data, not natural language or real retrieval.
- Perfect verifier with full access to ground truth.
- Deterministic programmatic agent, not a prompted or fine-tuned small model.
- Latency is Python function-call overhead only and does not estimate LLM/tool-call cost.
- The ledger-full condition did not add benefit beyond fact verification in this task family because arithmetic used repaired facts and was deterministic.

To overturn or strengthen this result, direct evidence would need to run the same ledger protocol with a real small LLM/tool-using agent, imperfect retrieval/verifier tools, held-out natural-language tasks, and an audit of whether the model produces sufficiently atomic falsifiable claims.

## Final decision

Decision: `finalize_negative` with `research_outcome: useful_signal`.

Stop rationale: bounded synthetic evidence supports the mechanism, but this is not a publication-grade validation of real small reasoning agents. The concrete next action is a bounded deepen follow-up using a real small open-weight or API model on natural-language tabular QA with noisy retrieval and the same ledger/no-ledger comparison.
