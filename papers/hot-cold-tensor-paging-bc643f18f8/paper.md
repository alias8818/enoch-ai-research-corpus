## 5. Results

### Key run-note evidence

- - True reusable mirror-buffer aliasing is technically possible in this bounded llama.cpp graph-lifecycle experiment: every run returned 0, and the pager reported `status=ready`, `tensors=480`, `swapped=480`, `skipped_layers=0`.
- - The reusable arena only needs about 312 MiB for the largest selected cold layer, so 512 MiB, 1 GiB, and 2 GiB arenas all exercised the same reuse behavior.
- - The performance result is negative for same-mechanism dynamic eviction/reuse: reusable pager throughput was only 3.28-3.30 tok/s for 8-token decode and 3.31 tok/s for 16-token decode, versus 5.65 and 5.81 tok/s static hot/cold baselines.
- - The reason is explicit in trace telemetry: per-token layer eviction/reload copied 84.0 GiB over 8 generated tokens and 168.0 GiB over 16 generated tokens, spending about 1.56 s and 3.10 s respectively inside pager copy work. Increasing arena budget did not help because the bottleneck is repeated cold-layer reload volume, not arena capacity.
- - Compared with the earlier persistent fixed-budget pager sweep, persistent consumed-tensor mirrors produced modest positive signals (+1% to +7% in bounded runs), while true reusable eviction/reuse lost about 42-43% versus static. This suggests the useful mechanism is selective persistent migration/residency, not naive decode-step eviction/reuse.
