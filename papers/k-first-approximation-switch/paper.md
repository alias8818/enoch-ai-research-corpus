## Abstract

We investigate an asymmetric KV-cache quantization policy—*K-First*—that allocates higher bit-width to cache keys while degrading values first under tight memory budgets. The motivating hypothesis is that keys govern attention routing while values carry answer payloads; therefore, over-quantizing keys should harm retrieval more than over-quantizing values. We test this in a controlled synthetic attention-cache benchmark where noisy queries retrieve a target span among 4,096 cached spans, isolating the proposed causal mechanism from confounds of full transformer inference. Across five random seeds (1,024 trials each, 5,120 total), the K4/V2 policy achieves 81.25% KV-footprint reduction relative to fp16 with only −0.59 percentage-point exact-match loss, outperforming the equal-budget uniform 3/3 policy by +2.54 points. At 75% reduction, K6/V2 matches or slightly exceeds uniform 4/4 quality. Conversely, value-first policies (K2/V4, K2/V6) collapse to approximately 59% accuracy despite identical total bit budgets. These results support the K-First switch in retrieval-heavy regimes, though we emphasize that this is a synthetic benchmark—not an end-to-end LLM evaluation—and full scientific closure requires real-model validation on downstream tasks.

## 1. Introduction

KV-cache memory is a primary bottleneck for long-context and high-throughput LLM inference. Prior work has established that asymmetric key/value treatment can yield gains: KIVI demonstrates 2.6× lower peak memory and 2.35×–3.47× throughput improvements via asymmetric KV quantization on real workloads (arXiv:2402.02750). AsymKV reports that transformer output loss is more sensitive to key quantization error than value quantization error (arXiv:2410.13212). The principle of allocating more bits to keys and fewer to values under fixed memory budgets has been articulated with configurations such as 4-bit keys paired with 2-bit values (arXiv:2502.15075). Industry guidance confirms that KV cache grows with context length and batch size, that int4 cache can largely preserve quality, and that int2 can degrade some long-context tasks (Hugging Face KV-cache quantization blog).

This work tests a specific operationalization of that principle: when the KV cache is under memory pressure, the system should *preserve key precision first and degrade value precision before key precision*. We call this the **K-First Approximation Switch**. The core causal claim is that key degradation harms attention routing (which span is selected) more than value degradation harms answer quality (what content is returned from the selected span), at least in retrieval-heavy workloads.

To isolate this mechanism before committing to a full inference-stack integration, we construct a synthetic benchmark that mimics the key-routing/value-retrieval structure of attention without the confounds of real transformer layers, multi-head interactions, and downstream task variance. This is a deliberate trade-off: internal validity is high, but external validity to real LLM workloads remains unproven.

## 2. Method

### 2.1 Benchmark Design

The benchmark simulates a retrieval scenario over a KV cache:

- **Cache structure:** 4,096 cached spans, each represented by a key vector and a value vector of dimension 64.
- **Query process:** A noisy query vector is generated by adding Gaussian noise (σ = 0.18) to a target key. The attention mechanism routes the query to the span with the highest key similarity (dot-product attention with score scale 24).
- **Answer extraction:** The value of the top-1 span is compared to the ground-truth value for exact match and top-1 span accuracy.

This design deliberately separates the *routing* function (keys determine which span is selected) from the *payload* function (values carry the answer), making the asymmetric sensitivity to key vs. value degradation directly observable.

### 2.2 Quantization Policies

We simulate uniform mid-rise quantization at specified bit-widths for keys and values independently. Dequantized tensors are used in the attention computation, approximating the behavior of real quantized kernels without requiring optimized CUDA implementations. The tested policies are:

| Policy | Key bits | Value bits | Total bits/KV pair | KV reduction vs. fp16 |
|---|---:|---:|---:|---:|
| fp16 reference | 16 | 16 | 32 | 0.00% |
| uniform_4_4 | 4 | 4 | 8 | 75.00% |
| k_first_6_2 | 6 | 2 | 8 | 75.00% |
| value_first_2_6 | 2 | 6 | 8 | 75.00% |
| uniform_3_3 | 3 | 3 | 6 | 81.25% |
| k_first_4_2 | 4 | 2 | 6 | 81.25% |
| value_first_2_4 | 2 | 4 | 6 | 81.25% |

The 75% reduction tier compares K-First, value-first, and uniform policies at the same total bit budget (8 bits per KV pair). The 81.25% tier does the same at a tighter budget (6 bits per KV pair).

### 2.3 Experimental Protocol

All experiments were run on an NVIDIA GB10 system (aarch64, Linux 6.17.0-1014-nvidia, approximately 128 GB RAM, no swap). Memory safety was monitored via `/proc/meminfo` and `/usr/bin/time -v` maximum RSS.

**Calibration:** 512 trials at n-ctx=4096, dim=64, classes=128, noise=0.18, with and without score-scale=24, to establish parameter sensitivity before the main run.

**Main experiment:** 5 independent seeds (10, 11, 12, 13, 14), each running 1,024 trials with n-ctx=4096, dim=64, classes=128, noise=0.18, score-scale=24. Results were appended to a single JSONL file and aggregated across all 5,120 trials per policy.

**Metrics:** Exact match (whether the dequantized value exactly matches the ground truth) and top-1 span accuracy (whether the top-1 attended span is the correct target span).

### 3.2 Analysis

**K-First outperforms equal-budget uniform at both compression tiers.** At 75% KV reduction, k_first_6_2 slightly exceeds uniform_4_4 on exact match (+0.06 points) and more clearly on top-1 span accuracy (+0.47 points). At 81.25% reduction, k_first_4_2 outperforms uniform_3_3 by +2.54 exact-match points and +2.63 top-1 points—a substantial gap at the same memory budget.

**Value-first policies collapse.** Both value_first_2_6 and value_first_2_4 fall to approximately 59% accuracy, a roughly 34-point drop from fp16. This confirms that in this retrieval-heavy regime, over-degrading keys destroys the routing mechanism, and the additional value precision cannot compensate. Notably, value_first_2_6 and value_first_2_4 produce nearly identical accuracy (~59.34%), suggesting that once key precision is reduced to 2 bits, additional value bits provide negligible benefit.

**K-First preserves routing fidelity.** The top-1 span accuracy of k_first_4_2 (93.59%) equals that of uniform_4_4 (93.59%) despite operating at a tighter 81.25% budget vs. 75%. This suggests that preserving key bits maintains the attention routing pathway even when value bits are aggressively reduced.

**Resource usage.** Maximum RSS per process was approximately 5.35 GB. The system had approximately 122 GB available at start with no swap, and no memory pressure was observed during any run.

### 3.3 Negative and Mixed Observations

- The advantage of k_first_6_2 over uniform_4_4 at 75% reduction is small (+0.06 exact-match points, +0.47 top-1 points). While directionally consistent, this margin may not be robust to different noise levels, dimensionalities, or task types.
- The benchmark does not reveal whether value degradation introduces subtle quality degradation in tasks where value precision matters (e.g., generation, summarization). The synthetic task's "exact match" criterion is binary and may not capture graded quality loss.
- Per-seed standard deviations are not reported in the aggregate summary, so formal confidence intervals cannot be constructed from the available data.

## 4. Limitations

1. **Synthetic benchmark, not end-to-end LLM evaluation.** The experiment uses a simplified attention model with independent key/value vectors and a single retrieval step. Real transformers have multi-head attention, residual connections, layer normalization, and multi-step interactions that may introduce value-sensitive failure modes not captured here. The results should not be generalized to real LLM workloads without validation.

2. **Fake quantization via numpy dequantization.** The quantization simulation uses mid-rise uniform quantization with dequantized tensors in standard floating-point arithmetic. This does not capture the latency, throughput, or numerical behavior of optimized integer kernels on GPU. Reported quality numbers are valid for the simulated quantization but should not be assumed to hold for hardware-quantized kernels without validation.

3. **No CPU/GPU offload or cache eviction modeling.** The benchmark does not simulate the traffic, thrash, or restore costs of a production inference stack with tiered cache (fp16 window + quantized tail + disk offload). The memory savings reported are theoretical bit-count reductions, not measured peak KV bytes in a serving system.

4. **Single task type.** The benchmark is retrieval-heavy. Tasks where value precision is critical—such as nuanced summarization, citation QA requiring exact phrasing, or generation tasks where the value vector carries fine-grained semantic content—may exhibit different sensitivity profiles. The K-First advantage may diminish or reverse for such tasks.

5. **No variance reporting across seeds.** The main summary reports means across 5,120 trials but does not include per-seed standard deviations. While the magnitude of the value-first collapse (~34 points) and the K-First advantage over uniform_3_3 (+2.54 points) is large enough to likely be robust, formal confidence intervals are not provided.

6. **Dimensionality and scale.** The benchmark uses dim=64 and 4,096 cached spans. Real LLM KV caches operate at much higher dimensions (e.g., 128–256 per head) and context lengths (tens to hundreds of thousands of tokens). Scaling behavior is untested.

## 6. Conclusion

In a controlled synthetic benchmark that isolates the key-routing/value-retrieval structure of attention, asymmetric KV-cache quantization that preserves keys and degrades values first (K-First) substantially outperforms both uniform quantization and value-first quantization at equal memory budgets. The K4/V2 configuration achieves 81.25% KV-footprint reduction relative to fp16 with only −0.59 percentage-point exact-match loss, and outperforms the equal-budget uniform 3/3 policy by +2.54 points. Value-first policies collapse to approximately 59% accuracy, confirming that key precision is critical for routing in retrieval-heavy regimes.

These results are consistent with prior literature reporting greater transformer sensitivity to key quantization than value quantization. However, this study remains a synthetic, single-task evaluation. The primary remaining scientific risk is whether real transformer layers and downstream tasks exhibit value-sensitive failure modes that this benchmark does not capture. We recommend next-stage validation on real inference stacks with LongBench, RULER, needle-in-a-haystack retrieval, citation QA, summarization, and extraction tasks, tracking quality, latency, cache traffic, and peak KV memory.

The operational recommendation for inference engineers, contingent on real-model validation, is: under KV-cache pressure, avoid K2-bit configurations for any span that must remain routable; prefer K4/V2 or K6/V2 for the compressed tail, with a residual fp16 or int8 window for recent tokens.
