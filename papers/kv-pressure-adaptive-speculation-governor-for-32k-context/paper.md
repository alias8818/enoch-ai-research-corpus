## Abstract

Speculative decoding accelerates autoregressive inference by drafting multiple tokens and verifying them in a single target-model forward pass, but its benefit depends on both draft acceptance rate and the memory headroom available for draft-token KV cache entries. At long context lengths (>32k tokens) and high batch sizes, KV cache pressure can cause fixed-budget speculative decoding to regress below baseline autoregressive throughput. We propose and analytically evaluate a runtime governor that dynamically adjusts speculative draft budget (K) and method selection based on observed KV cache pressure and recent speculative acceptance rate. In a parameterized simulation modeling a Llama-3.x-8B-class GQA KV cache across 54 scenarios (context 8k–64k, batch 1–8, temperature 0.2–0.8, domains code/logs/chat), the adaptive governor eliminates all regression cases that affect fixed K=8 speculation (18 regressions, worst case 0.696× baseline) and static context-aware K reduction (5 regressions, worst case 0.928× baseline), while retaining a geometric-mean speedup of 1.369× over autoregressive decoding. This robustness comes at the cost of lower headline speedup than the static context policy (1.445× geomean), which still regresses in 5 scenarios. These results are from an analytical simulator, not from a live model-serving framework; we do not claim production validation. We present sensitivity results across KV budgets (48–96 GiB) and outline the benchmarking required for scientific closure.

## 1. Introduction

Speculative decoding accelerates autoregressive language model inference by using a smaller draft model or a self-speculative method to propose multiple candidate tokens, which the target model then verifies in a single forward pass. When the draft acceptance rate is high, this yields substantial wall-clock speedups. However, the technique introduces additional KV cache entries for draft tokens—entries that must be allocated before verification and rewound if rejected.

At short context lengths and low batch sizes, the KV cache overhead of speculative tokens is modest. As context length grows beyond 32k tokens and batch sizes increase, the KV cache consumes a large fraction of available memory. In this regime, speculative draft tokens compete with base-sequence KV entries for the same memory budget, and the scheduler may face pressure that degrades throughput or causes out-of-memory conditions.

Current serving frameworks expose some of the necessary telemetry. vLLM reports `kv_cache_usage_perc` and `gpu_cache_usage_perc` as Prometheus-style metrics. TensorRT-LLM explicitly counts draft-token pages toward scheduler token limits and rewinds rejected-token pages after verification. SGLang provides OOM-mitigation guidance that includes reducing draft tree size and limiting concurrent requests. However, existing speculative decoding deployments typically use a fixed draft budget (K) or at best a static context-length-based reduction. Neither approach accounts for the dynamic interplay between current KV pressure, recent acceptance rate, and the marginal value of additional speculative tokens.

We investigate whether a runtime governor that observes KV pressure and recent speculative acceptance can adaptively choose K (and, by extension, draft method or tree budget) to avoid the regressions of fixed speculative decoding while retaining most of the speedup when speculation is beneficial. We evaluate this question through an analytical simulation and a literature-informed design argument, and we report both positive and negative findings.

## 2. Method

### 2.1 KV Cache Model

We model a Llama-3.x-8B-class architecture with grouped-query attention (GQA):

- Layers: 32
- KV heads: 8
- Head dimension: 128
- KV dtype: fp16/bf16 (2 bytes per element)

KV bytes per token per sequence:

$$B_{\text{token}} = 2 \times L \times H_{\text{kv}} \times D \times 2 = 2 \times 32 \times 8 \times 128 \times 2 = 131{,}072 \text{ bytes} \approx 128 \text{ KiB}$$

where the leading factor of 2 accounts for K and V projections. Total KV memory for a sequence of length $S$ tokens at batch size $N$ is:

$$M_{\text{KV}} = N \times S \times B_{\text{token}}$$

### 2.2 Scenario Matrix

We evaluate across 54 scenarios formed by the Cartesian product of:

- Context length: 8k, 32k, 64k tokens
- Batch size: 1, 4, 8
- Temperature: 0.2, 0.8
- Domain: code, logs, chat

Domain affects the simulated acceptance rate: code domains yield higher acceptance (more deterministic outputs), while chat at high temperature yields lower acceptance.

### 2.3 Policies Compared

**Target-only AR.** Baseline autoregressive decoding with no speculation. Speedup ratio = 1.0 by definition.

**Fixed K=8.** Standard speculative decoding with a fixed draft budget of 8 tokens. This represents the common deployment default.

**Static context-aware K reduction.** A policy that reduces K based on context length alone (e.g., K=8 for ≤8k, K=4 for 32k, K=2 for 64k), without observing dynamic pressure or acceptance.

**Adaptive pressure/acceptance governor.** The proposed policy. Before each scheduling decision, the governor observes:

1. Current KV cache utilization fraction (pressure).
2. Exponential moving average (EMA) of recent speculative acceptance rate.
3. Current context length and batch size.
4. Sampling temperature.

The governor then selects K from {1, 2, 4, 8} and may switch between draft methods (e.g., full speculative vs. suffix/ngram mode). When KV pressure exceeds a threshold or acceptance EMA falls below a threshold, the governor reduces K or disables speculation entirely (K=1, effectively autoregressive).

### 2.4 Simulation Implementation

The simulation was implemented in `scripts/kv_spec_governor_sim.py` and executed on a GB10-equipped host (Linux aarch64, 121 GiB RAM, Python 3.12.3). The simulator computes KV memory occupancy, estimates acceptance rates per domain/temperature, and models the throughput effect of speculation as a function of acceptance rate, K, and the memory overhead of draft tokens. It does not execute model inference; it is a parameterized analytical model, not a live serving benchmark.

The primary KV budget was set to 80 GiB, with sensitivity runs at 48, 64, and 96 GiB.

### 2.5 Speedup Model

For a given scenario, the simulated speedup over autoregressive decoding is modeled as:

$$\text{speedup}(K, \alpha, p) = f(K, \alpha) \times g(p)$$

where $\alpha$ is the acceptance rate, $p$ is the KV pressure fraction, $f$ captures the speculative acceleration as a function of K and acceptance, and $g$ is a penalty function that reduces effective throughput as pressure approaches capacity. When $g(p)$ is sufficiently degraded, the model predicts that speculation harms throughput relative to AR, producing a regression. The functional forms of $f$ and $g$ are parameterized estimates that require calibration against measured data.

## 3. Results

### 3.1 Core Results at 80 GiB KV Budget

| Policy | Geomean Speedup | Mean Speedup | Regression Cases | OOM-Risk Cases | Min Speedup |
|---|---:|---:|---:|---:|---:|
| Target-only AR | 1.000 | 1.000 | 0 | 0 | 1.000 |
| Fixed K=8 | 1.225 | 1.306 | 18 | 0 | 0.696 |
| Static context | 1.445 | 1.502 | 5 | 0 | 0.928 |
| Adaptive pressure | 1.369 | 1.431 | 0 | 0 | 1.000 |

Over 54 scenarios:

- The adaptive governor beats fixed K=8 in 30 of 54 cases.
- The adaptive governor beats static context in 13 of 54 cases.
- The static context policy achieves a higher geometric-mean speedup (1.445× vs. 1.369×) but regresses in 5 scenarios; the adaptive governor trades headline speed for zero regressions.

### 3.2 Regression Analysis

The 18 regression cases under fixed K=8 are concentrated in long-context, high-temperature, low-determinism scenarios. The worst case occurs at 64k context, batch 1, temperature 0.8, chat domain, where fixed K=8 yields only 0.696× the throughput of autoregressive decoding—a 30.4% slowdown.

The 5 regression cases under the static context policy occur at the boundary where context-length-based K reduction is insufficient to account for batch-size-induced pressure. The worst case is 0.928× baseline.

The adaptive governor achieves a minimum speedup of 1.000× (i.e., it falls back to autoregressive decoding rather than regressing) by reducing K to 1 when conditions predict harm.

### 3.3 High-Pressure Regime Behavior

At 64k context with batch 8 under the 80 GiB budget, base KV pressure reaches approximately 0.80 before any speculative extras are added. In this regime, the adaptive governor switches to K=1–2 suffix/ngram mode even when acceptance rate is favorable, because the dynamic headroom for additional draft-token KV entries is already severely constrained. This is the intended conservative behavior: the governor recognizes that the marginal memory cost of speculation exceeds its benefit under high pressure.

### 3.4 Sensitivity to KV Budget

Sensitivity runs were conducted at 48, 64, 80, and 96 GiB effective KV budgets. At lower budgets (48 GiB), the high-pressure regime begins at shorter context lengths and lower batch sizes, causing the adaptive governor to disable speculation more frequently and reducing its average speedup. At higher budgets (96 GiB), more headroom is available and the adaptive governor approaches the static context policy's speedup while still maintaining zero regressions. The detailed per-budget breakdowns are available in the referenced result files.

### 3.5 Mixed and Negative Findings

The adaptive governor does not dominate the static context policy on average speedup. In 41 of 54 scenarios, the static context policy equals or exceeds the adaptive governor's speedup. The adaptive governor's advantage is entirely in robustness: it eliminates regressions that the static policy incurs. Whether this tradeoff is favorable depends on deployment requirements. For latency-sensitive serving where tail performance matters, zero regressions may justify lower average speedup. For throughput-oriented batch processing where average speed dominates, the static policy may be preferable despite occasional regressions.

Additionally, the adaptive governor's benefit over fixed K=8 is concentrated in the long-context/high-batch tail. At 8k context with batch 1, all three speculative policies perform similarly, as KV pressure is low and acceptance dominates.

## 4. Limitations

1. **Analytical simulation, not measurement.** All results derive from a parameterized simulator (`kv_spec_governor_sim.py`) that models KV cache occupancy and estimates throughput effects. No real model-serving framework (vLLM, SGLang, TensorRT-LLM) was installed or benchmarked in this run. The simulator's acceptance-rate estimates, pressure-to-throughput penalty functions, and speedup models require calibration against measured data.

2. **No framework integration.** The governor logic has not been implemented as a hook or scheduler modification in any serving framework. Integration overhead, latency of telemetry reads, and scheduler interaction effects are not modeled.

3. **Simplified speedup model.** The relationship between KV pressure, acceptance rate, K, and effective throughput is modeled with parameterized functions rather than measured. Real systems exhibit non-linear scheduling effects, preemption, swapping, and memory fragmentation that the simulator does not capture.

4. **Single architecture class.** The KV cache model targets a Llama-3.x-8B-class GQA configuration (32 layers, 8 KV heads, head dimension 128, fp16/bf16). Other architectures (e.g., MHA models, larger parameter counts, different head configurations) will have different KV-per-token costs and different pressure profiles.

5. **No end-to-end latency metrics.** The simulator reports relative throughput speedup but does not model time-to-first-token (TTFT), inter-token latency (ITL), or scheduling overhead. These metrics are critical for interactive serving.

6. **GB10 memory accounting limitation.** The GB10 GPU reports memory accounting as "Not Supported" via `nvidia-smi`. The simulation relies on system `MemAvailable` and a unified memory architecture (UMA) posture rather than discrete GPU memory fields. Real deployments may need different telemetry paths.

7. **Acceptance rate modeling.** Domain and temperature effects on acceptance rate are estimated rather than measured. Real acceptance rates depend on the specific draft/target model pair, prompt distribution, and sampling implementation.

8. **Claim audit status.** The claim ledger for this artifact is currently empty with audit status "blocked_empty_claims." No structured claims have passed evidence audit. The findings reported here should be treated as preliminary simulation observations pending formal claim extraction and verification.

## 6. Conclusion

We have presented and analytically evaluated a KV-pressure adaptive speculation governor for long-context speculative decoding. The central finding is a robustness–speedup tradeoff: the adaptive governor eliminates all simulated regression cases (0 regressions across 54 scenarios) while retaining a geometric-mean speedup of 1.369× over autoregressive decoding. This contrasts with fixed K=8 speculation (1.225× geomean, 18 regressions, worst case 0.696×) and static context-aware K reduction (1.445× geomean, 5 regressions, worst case 0.928×).

The adaptive governor does not achieve the highest average speedup; the static context policy outperforms it in 41 of 54 scenarios on a per-scenario basis. The adaptive governor's contribution is entirely in the tail: it prevents speculation from harming throughput when KV pressure is high and acceptance is low. Whether this tradeoff is worthwhile depends on whether a deployment can tolerate occasional regressions in exchange for higher average speed.

These results are from an analytical simulation and have not been validated against a live model-serving framework. The simulator's parameterized models of acceptance rates, pressure penalties, and throughput effects require calibration. We identify the following as necessary for scientific closure:

1. Installation of a serving framework (SGLang or vLLM) on the target GB10 hardware.
2. Calibration of acceptance rates and throughput effects against measured data across the scenario matrix.
3. Implementation of the governor as a scheduler hook or wrapper that reads framework KV metrics and adjusts K before each scheduling decision.
4. End-to-end benchmarking with TTFT, ITL, and throughput metrics across the full context/batch/temperature/domain matrix.

The governor concept is viable for next-stage implementation. The analytical evidence supports the hypothesis that KV-pressure-aware speculation control can serve as a robustness layer for long-context speculative decoding, but this hypothesis remains unconfirmed by real serving benchmarks.
