## Abstract

We investigate whether a Toolformer-style self-supervised gate can learn to route factual questions to an external lookup tool while suppressing tool calls when the answer is already available in-context. Using a dependency-light synthetic benchmark—a character-trigram Naive Bayes parametric answerer paired with a logistic-regression gate over a deterministic entity-registry table—we show that the gated system improves out-of-distribution (OOD) factual lookup accuracy from near chance (0.005 ± 0.007) to 0.932 ± 0.116 across 10 random seeds, while maintaining a zero tool-call rate on control prompts where the answer token is visible in-context. Overall gated accuracy reaches 0.967 ± 0.056 versus 0.526 ± 0.009 for the parametric-only baseline. These results support the narrow claim that simple self-supervised tool-call labels can teach a small gate to selectively invoke external lookup, but the synthetic setting, deterministic tool, and hand-written parsing substantially limit generalizability.

## Introduction

Language models that can invoke external tools—retrievers, calculators, code interpreters—offer a path to improved factual accuracy without scaling parametric memory. The Toolformer framework proposes that models can learn *when* to call tools by generating self-supervised training signals: insert a tool call, observe whether the result improves the answer, and use this as a label for tool-use decisions.

A central tension in such systems is selectivity. Indiscriminate tool invocation wastes latency and cost; failure to invoke when needed leaves factual gaps. An ideal gate calls the tool exactly when the parametric model's answer is wrong and the tool can correct it, and abstains when the answer is already available in-context.

We study this question in a controlled toy setting that strips away the complexity of end-to-end transformer training to isolate the gating problem. Our contributions are:

1. A synthetic registry task where entity names map to random access-code labels, with a controlled train/test entity split that makes OOD factual generalization near chance for the parametric answerer.
2. A self-supervised labeling procedure that marks a tool call as positive when the parametric answer is incorrect and the lookup result corrects it, and negative otherwise (including on control prompts where the answer is in-context).
3. Empirical evidence over 10 seeds that a small logistic gate trained on these labels achieves high OOD lookup accuracy (0.932 ± 0.116) while making zero tool calls on control prompts.

We emphasize that this is a toy experiment with a deterministic lookup tool, hand-written parsing, and a non-neural parametric answerer. The results establish a viability signal for the self-supervised gating principle but do not demonstrate scalability or robustness in realistic settings.

## Method

### Task Design

We construct a synthetic registry of entity–value pairs, where each entity name maps to a unique label of the form `VAL_###`. The task is to answer two types of prompts:

- **Lookup prompts**: "What is the access code for [entity]?" — the answer is the entity's registry value, which must be retrieved from the lookup table.
- **Control prompts**: "The access code for [entity] is [value]. What is the access code for [entity]?" — the answer token is explicitly present in the prompt context.

Training data covers only 55% of entities. Test data includes lookup and control prompts for both seen and unseen entities, enabling measurement of OOD generalization.

### Parametric Answerer

We use a character-trigram Naive Bayes classifier as the parametric answerer, trained exclusively on question–answer pairs from the training entities. This model has no access to the lookup tool and represents a lightweight, non-neural baseline whose OOD generalization is expected to be near chance, since the synthetic task assigns random values to unseen entities with no exploitable semantic structure.

### Lookup Tool

A deterministic `lookup(entity)` function returns the correct value for any entity present in the full registry table. The tool always succeeds for lookup prompts (all queried entities have table entries) and is never needed for control prompts (the answer is in-context). This oracle-like property of the tool is an intentional simplification that establishes an upper bound but does not model retrieval noise, missing entries, or latency.

### Self-Supervised Gate Training

Following the Toolformer principle, we generate calibration traces by running the parametric answerer on a held-out calibration set and comparing its predictions to ground truth:

- **Positive label**: the parametric answer is wrong, and inserting the lookup result yields the correct answer.
- **Negative label**: the parametric answer is already correct (including all control prompts, where the in-context answer makes the parametric model correct by construction).

A logistic regressor is trained on these labels using features derived from the input. The gate outputs a probability of tool-call utility, thresholded at 0.5.

### Experimental Configuration

**Main run.** 10 seeds (0–9), 500 entities, 96 distinct values, 55% training entity fraction, 500 calibration entities, 400 gate training epochs, learning rate 0.3, trigram order 3, smoothing α = 0.1, decision threshold 0.5.

**Smoke test.** 1 seed (0), 80 entities, 32 values, same hyperparameters otherwise.

**Low-calibration ablation.** 5 seeds (0–4), 500 entities, 96 values, 25 calibration entities (reduced from 500), remaining hyperparameters unchanged.

## Results

### Main Run (10 Seeds, 500 Entities, 96 Values)

| Metric | Mean | Std |
|---|---:|---:|
| Parametric-only accuracy | 0.5261 | 0.0089 |
| Always-lookup accuracy | 1.0000 | 0.0000 |
| Gated lookup-aware accuracy | 0.9674 | 0.0555 |
| Gated tool-call rate | 0.5112 | 0.0863 |
| Gated control tool-call rate | 0.0000 | 0.0000 |
| OOD lookup parametric accuracy | 0.0053 | 0.0065 |
| OOD lookup gated accuracy | 0.9320 | 0.1160 |

The parametric-only answerer achieves 52.6% overall accuracy, reflecting its ability to answer seen-entity lookup questions and all control questions correctly, but near-zero accuracy (0.5%) on OOD entity lookups. The always-lookup oracle achieves perfect accuracy by construction: all lookup prompts have table entries, and control prompts are parseable without tool use.

The gated system achieves 96.7% overall accuracy, a 44.1 percentage-point improvement over the parametric baseline. On OOD lookup questions specifically, the gate improves accuracy from 0.5% to 93.2%, an absolute gain of +0.927. The gate calls the tool on 51.1% of test examples, roughly matching the fraction of lookup prompts in the test set, and makes **zero** tool calls on control prompts across all 10 seeds (std = 0.0000).

### Smoke Test (1 Seed, 80 Entities, 32 Values)

The smoke test confirmed the pipeline on a smaller configuration. Results: parametric accuracy 0.518, always-lookup accuracy 1.0, gated accuracy 1.0, gated tool-call rate 0.566, gated control tool-call rate 0.0, OOD parametric accuracy 0.028, OOD gated accuracy 1.0. Elapsed time 0.047s. The smoke test produced ceiling performance on this small configuration, which is consistent with the gate having an easier separation problem at 80 entities.

### Seed Variability

The standard deviation of OOD gated accuracy (0.116) is notably higher than for overall accuracy (0.056), indicating that some random seeds produce weaker gates. This variability is consistent with the gate being a small logistic regressor trained on limited features; some calibration splits yield less separable positive/negative examples. This is a meaningful signal rather than mere noise: it highlights the sensitivity of self-supervised gating to calibration data quality and the fragility of a minimal gate architecture.

## Limitations

1. **Synthetic task.** The registry task uses random entity–value mappings with no semantic structure. Real factual QA involves compositional, relational, and ambiguous queries that are substantially harder for both the parametric answerer and the gate.

2. **Deterministic lookup tool.** The tool always returns the correct answer for any queried entity. Real retrieval systems introduce noise (wrong passages, stale data, missing entries) and latency/cost tradeoffs that the gate must also model. The always-lookup upper bound of 1.0 is an artifact of this design.

3. **Hand-written parsing.** Entity extraction and tool-call insertion use hand-written rules. An end-to-end system must learn to identify tool-call positions and arguments from data, which introduces additional failure modes.

4. **Non-neural parametric answerer.** The character-trigram Naive Bayes classifier is intentionally simple. Whether a neural language model produces equally separable self-supervised labels for gate training remains an open question. The error profile of a neural model may differ substantially, potentially making the positive/negative label boundary less clean.

5. **Oracle-like always-lookup upper bound.** The perfect always-lookup accuracy is an artifact of the toy design: all lookup entities have table entries and controls are trivially parseable. In realistic settings, always-lookup would incur cost, latency, and retrieval errors, making it a non-trivial baseline rather than a ceiling.

6. **Gate variability.** The 0.116 standard deviation on OOD gated accuracy indicates that the gate is sensitive to random seed and calibration split. Some seeds produce substantially weaker gates. Stronger gating architectures or threshold tuning may be necessary for reliable deployment.

7. **No cost modeling.** The gate optimizes for accuracy only. A production system must trade off accuracy against tool-call cost, latency, and failure modes.

## Conclusion

In a controlled synthetic setting, a Toolformer-style self-supervised gate trained on calibration traces of a parametric answerer can learn to selectively invoke an external lookup tool, improving OOD factual accuracy from near chance (0.005) to 0.932 while making zero spurious tool calls on in-context control prompts. This result is bounded: the task is synthetic, the tool is deterministic, parsing is hand-written, and the parametric model is a simple classifier. The observed seed variability in gate performance (OOD std = 0.116) further cautions against overgeneralization. Nevertheless, the core finding—that self-supervised error-correction labels can produce a selective tool-use gate even with a minimal learning architecture—provides a focused viability signal for the Toolformer gating principle and motivates future work on noisy tools, learned action spans, cost-aware gating, and neural parametric models.
