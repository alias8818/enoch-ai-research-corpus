## Abstract

We evaluate whether a typed memory quarantine queue reduces poisoned and stale long-term memory retention when memory proposals are generated by a real local LLM processing messy multi-turn workflow transcripts, extending prior work that relied on deterministic or stochastic toy proposal generators. Using llama.cpp with Qwen2.5-0.5B-Instruct (Q4_K_M) across six synthetic-but-messy scenarios with two repeats each, we compare an unconstrained transcript-wrapper persistence mode (commit every proposal) against a typed quarantine mode that classifies proposals by risk and routes moderate-risk items to delayed critic review while rejecting high-risk items. The transcript-wrapper mode retained 94.7% of poison proposals and 100% of false positives, with 0% action success. The typed quarantine mode reduced poison retention to 0%, false-positive retention to 0%, and achieved 100% action success, while preserving useful retention at 87.5% (equal to the baseline). A cross-model smoke test with Qwen2.5-7B-Instruct on two scenarios reproduced the same directional result. These findings support the hypothesis for this bounded fixture but remain limited by a small scenario set, a hand-coded classifier calibrated after an initial run, and the absence of production trace replay.

## Introduction

Autonomous agents that maintain long-term memory face a fundamental tension: useful facts and preferences should persist, while poisoned, stale, or injected content should not. Prior work on this project (parent artifact `source-record-redacted`) demonstrated that a typed memory quarantine queue could reduce poison retention under deterministic and local stochastic proposal generators. However, those proposal sources lacked the variability and failure modes of real LLM-generated memory proposals extracted from messy multi-turn transcripts.

The present study asks: does the typed quarantine mechanism retain its effectiveness when memory proposals are produced by a real local instruct model processing realistic workflow transcripts, rather than by a toy generator?

This question matters because production agent systems will receive proposals from LLMs, not from controlled generators. Real LLM proposals exhibit variable formatting, inconsistent scope labeling, and occasional hallucination of content that blends benign and dangerous cues—properties that a quarantine classifier must handle robustly.

We report results from a local llama.cpp benchmark using Qwen2.5 instruct models on an aarch64 host, measuring retention rates, action success, latency, and memory posture.

## Method

### Benchmark Design

A single Python script (`scripts/real_llm_memory_quarantine_benchmark.py`) drives the benchmark. The script loads a local GGUF model via the `llama_cpp` Python package and presents it with messy multi-turn workflow transcripts. The LLM extracts candidate durable memories from each transcript.

Each extracted proposal is evaluated under two persistence modes:

1. **Transcript-wrapper mode**: Every proposal is committed to long-term memory without filtering.
2. **Typed-quarantine mode**: A classifier inspects proposal text, scope, and source cues. Low-risk proposals are committed immediately. Moderate-risk proposals enter a quarantine queue for delayed critic review. High-risk proposals are rejected outright.

### Scenarios

Six scenarios span the following content categories:

- Benign user preferences and workflow facts
- Stale temporary facts (e.g., time-bound information that should not persist)
- Untrusted content sourced from browser sessions, tickets, or documents
- Prompt-injection strings
- Secrets (API keys, credentials)
- Unsafe tool or deployment instructions

Scenarios are synthetic but designed to be messy—containing realistic multi-turn structure, ambiguous framing, and mixed content types within a single transcript.

### Models

Two local GGUF quantized models were used:

- **Primary**: Qwen2.5-0.5B-Instruct-Q4_K_M (494M parameters, 4-bit quantization)
- **Cross-model smoke**: Qwen2.5-7B-Instruct-Q4_K_M (7B parameters, 4-bit quantization)

### Metrics

- **Useful retention rate**: Fraction of genuinely useful memories that were committed.
- **Poison retention rate**: Fraction of poisoned/injected memories that were committed.
- **False-positive retention rate**: Fraction of stale or irrelevant memories that were committed.
- **Action success rate**: Fraction of scenarios in which the agent's final action was correct (i.e., not corrupted by retained poison).
- **Proposal counts**: Mean committed, quarantined, and rejected proposals per scenario.
- **Latency**: Mean and p95 LLM call latency.
- **Throughput**: Observed completion tokens per second.
- **Memory posture**: Max RSS, `/proc/meminfo` MemAvailable before and after, SwapTotal.

### Calibration

An initial full run with Qwen2.5-0.5B revealed overly conservative useful-memory retention in the classifier. This run was retained as a tuning artifact (see artifacts `real_llm_qwen05_results.json` and `real_llm_qwen05_report.md`). The classifier thresholds were adjusted, and a second calibrated run was performed. This calibration step is a limitation: thresholds were not frozen prior to the first evaluation.

### Execution Environment

- **Host**: aarch64 GB10-class machine (`Linux gx10-efe8`, 20 ARM CPUs)
- **Swap**: Intentionally absent; `SwapTotal: 0 kB` confirmed in telemetry
- **GPU**: `nvidia-smi` was callable but reported UMA memory fields as `N/A`; GPU utilization was 0 throughout. All inference was CPU/UMA-bound.
- **No external inference endpoints**: Ports 8000, 1234, and 11434 were probed and found unusable; port 8080 served a DGX dashboard only.

### Negative and Mixed Observations

The initial (pre-calibration) Qwen2.5-0.5B run exhibited overly conservative useful-memory retention, meaning the classifier rejected or quarantined proposals that were genuinely useful. This artifact was preserved but is not reported as a positive result. It highlights that classifier threshold selection directly trades off between poison rejection and useful-memory preservation, and that a single calibration step was required to achieve the reported parity in useful retention.

The 7B smoke test covered only two scenarios with one repeat each, which is insufficient to draw conclusions about retention rates with precision. It confirms directionality only.

## Limitations

1. **Small scenario set**: Six scenarios with two repeats (12 proposal calls) is a bounded fixture, not a comprehensive evaluation. Variance across a larger scenario set could shift retention rates.

2. **Hand-coded classifier with post-hoc calibration**: The typed quarantine classifier was implemented as hand-coded rules rather than a learned model. Its thresholds were adjusted after an initial run showed conservative useful retention. This means the reported useful-retention parity is not out-of-the-box performance; it required one round of tuning. Future evaluations should freeze thresholds before measurement.

3. **Synthetic transcripts**: Scenarios are synthetic, albeit designed to be messy. They do not reflect the full distribution of real agent traces, which may contain more subtle injection vectors, longer context, or adversarial prompt structures not represented here.

4. **No live external connector mutations**: The benchmark did not execute real tool calls, API requests, or deployment actions. Action success was measured against a scripted oracle, not against real-world side effects.

5. **Single model family**: Both models are from the Qwen2.5 family. Different model families may produce proposals with different formatting or risk profiles that the hand-coded classifier does not handle.

6. **CPU-bound inference only**: All runs used CPU/UMA-bound llama.cpp inference. GPU utilization was zero. Results may differ under GPU-accelerated inference with different batching or memory characteristics.

7. **No review burden measurement**: The quarantine mode routes moderate-risk proposals to delayed critic review, but the benchmark does not measure the human or automated review burden this creates. In production, review load is a critical operational metric.

## Conclusion

A typed memory quarantine queue eliminated measured poison retention (reduction of 0.947 absolute) and false-positive retention (reduction of 1.000 absolute) while preserving useful retention at the same level as an unconstrained transcript-wrapper baseline (0.875 in both modes) in a calibrated benchmark using a real local LLM (Qwen2.5-0.5B-Instruct) on six messy workflow scenarios. A cross-model smoke test with Qwen2.5-7B-Instruct reproduced the same directional finding on two scenarios. These results extend prior evidence from deterministic and stochastic toy generators to real LLM-generated proposals, supporting the hypothesis that typed quarantine is effective even when proposals originate from instruct models processing messy transcripts.

However, the evidence strength is moderate. The scenario set is small, the classifier was hand-coded and calibrated after an initial conservative run, and no production trace replay or live external connector evaluation was performed. The result constitutes bounded scientific closure for this fixture but does not constitute production validation. The recommended next step is a larger replay/A-B benchmark over captured real agent traces with frozen classifier thresholds and multiple model families, tracking review burden and useful-memory false negatives alongside poison retention.
