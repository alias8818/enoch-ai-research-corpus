## Goal

Test whether persisted per-tensor 8-bit SGDM momentum remains competitive with fp32 SGDM momentum across fixed seeds on a GPT-2-small LoRA language-modeling task, while adding a modern AdamW baseline and a no-momentum SGD control.

## Experimental design

- Model: Hugging Face `gpt2` / GPT-2-small, frozen base weights.
- Adapter: LoRA rank 8 on each GPT-2 attention `c_attn` and `c_proj` Conv1D module.
- Datasets: Alice in Wonderland from Project Gutenberg, Penn Treebank from the Zaremba split, and Tiny Shakespeare.
- Optimizers: no-momentum SGD, fp32 SGDM, persisted 8-bit SGDM, and AdamW.
- Direct metrics: aggregate and per-dataset validation cross-entropy/perplexity, paired seed deltas, optimizer-state bytes, throughput, and memory snapshots.
- Fixed seeds planned for validation: `17`, `23`, `31`.

## Interpretation

The bounded mechanism is supported in the tested scope. Across three fixed seeds and three real text datasets, persisted per-tensor uint8 SGDM momentum matched fp32 SGDM validation loss within `0.00231` loss on every seed while using about 25% of fp32 SGDM momentum-state bytes. The no-momentum SGD control was materially worse on every seed, so the result is not explained by momentum being irrelevant. The AdamW baseline, with the configured `1e-3` LoRA learning rate and weight decay `0.01`, was also worse than both SGDM variants on all three seeds, so this run does not collapse once a modern optimizer baseline is included.

This is still not publication-ready. The evidence is direct and multi-seed, but it is limited to GPT-2-small, LoRA-only adaptation, three small public corpora, one LoRA rank, one SGDM/AdamW hyperparameter setting, and a Python optimizer implementation rather than a production blockwise/fused optimizer. A broader paper claim would need stronger task coverage, tuned AdamW/SGDM sweeps, longer-horizon and resume/persistence stress tests in the same multi-dataset setup, and a production implementation with accurate end-to-end memory accounting.
