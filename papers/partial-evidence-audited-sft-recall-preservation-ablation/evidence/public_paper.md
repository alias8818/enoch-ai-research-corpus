## Abstract

We report results from a controlled LoRA supervised fine-tuning (SFT) ablation on `microsoft/Phi-4-mini-instruct` that targets the reduction of partial-refusal over-answer behavior while preserving full-evidence answer recall. The ablation trains a rank-8 LoRA adapter on 240 audited training examples—comprising partial-refusal hard negatives and full-answer recall-preservation examples—and evaluates on 72 held-out rows stratified by domain and set. On the held-out evaluation, partial-refusal over-answer decreased from 0.333 to 0.000 (a 33.3 percentage-point reduction), and full-evidence answer recall increased from 0.750 to 1.000 (a 25.0 percentage-point improvement). Naive-missing over-answer remained unchanged at 0.0 percentage-point delta. The branch-specific kill condition (requiring ≥10 pp reduction in partial-refusal over-answer with ≤5 pp full-evidence recall loss) was not triggered. However, these results come from a single seed, a single model, and a small held-out evaluation set; confidence is assessed as medium and evidence strength as moderate. Replication with multiple seeds and larger held-out splits is recommended before treating the finding as robust.

## 1. Introduction

Language models sometimes produce partial-refusal over-answers: responses that partially address a query while embedding unnecessary refusal language, even when the model possesses sufficient evidence to answer fully. This behavior is distinct from appropriate refusal on harmful queries and from naive-missing over-answers where the model provides extraneous information without refusal framing.

A natural corrective approach is supervised fine-tuning on audited hard negatives—examples where the model exhibited partial-refusal over-answer—alongside recall-preservation examples where the model answered correctly. The central question is whether such targeted SFT can reduce the undesirable partial-refusal cluster without degrading the model's ability to answer fully when evidence is available.

This report documents a concrete ablation test of that hypothesis on `microsoft/Phi-4-mini-instruct` using a rank-8 LoRA adapter, a human-graded hardening corpus, and exact-match/refusal scoring. The project was governed by a pre-registered kill condition: the ablation would be classified as negative if partial-refusal over-answer did not decrease by at least 10 percentage points, or if full-evidence answer recall decreased by more than 5 percentage points relative to the base model.

## 2. Method

### 2.1 Data

The training and evaluation data derive from a human-graded hardening set (`data/human_graded_hardening_set.jsonl`) and an audit queue (`data/human_audit_queue.csv`), both acquired from a parent project. Each example is labeled as one of: `full_answer` (correct, complete response), `partial_refusal` (response contains unnecessary refusal language despite sufficient evidence), or `naive_missing` (response provides extraneous information without refusal framing).

The SFT harness (`src/sft_lora_ablation.py`) holds out the first N rows per domain and set for evaluation, ensuring stratification. Training uses only audited `partial_refusal` hard negatives and `full_answer` recall-preservation examples. The held-out evaluation set is scored with the same exact-match and refusal detection logic used by the parent benchmark (`src/hardening_benchmark.py`, `src/live_model_hardening_eval.py`).

### 2.2 Model and Training

- **Base model:** `microsoft/Phi-4-mini-instruct`
- **Adapter:** LoRA, rank 8
- **Training rows:** 240
- **Evaluation rows:** 72 (4 held out per domain/set)
- **Training steps:** 80 microsteps
- **Training wall time:** 21.36 s
- **Throughput:** 3.74 steps/s, 7.49 samples/s

### 2.3 Evaluation Protocol

The base model and the LoRA-adapted model are both evaluated on the same 72 held-out rows. Three metrics are computed:

1. **Partial-refusal over-answer rate:** fraction of held-out examples where the model produces a partial-refusal response.
2. **Full-evidence answer recall:** fraction of held-out examples where the model produces a correct full answer.
3. **Naive-missing over-answer rate:** fraction of held-out examples where the model produces a naive-missing over-answer.

### 2.4 Kill Condition

The ablation is classified as negative if either:
- Partial-refusal over-answer reduction < 10 percentage points, or
- Full-evidence answer recall decrease > 5 percentage points relative to the base model.

### 2.5 Smoke Test

A 2-step correctness smoke (`results/phi_lora_sft_smoke`) was run over 18 held-out examples to verify loading, generation, PEFT saving, scoring, and telemetry before the main ablation. As expected for a 2-step smoke, metric deltas were 0 pp.

## 3. Results

### 3.1 Main Ablation Metrics

| Metric | Base Model | LoRA Adapter | Delta (pp) |
|---|---|---|---|
| Partial-refusal over-answer rate | 0.333 | 0.000 | −33.3 |
| Full-evidence answer recall | 0.750 | 1.000 | +25.0 |
| Naive-missing over-answer rate | — | — | 0.0 |

The partial-refusal over-answer rate decreased by 33.3 percentage points, exceeding the 10 pp threshold. Full-evidence answer recall increased by 25.0 percentage points, well within the ≤5 pp loss tolerance (in fact improving). The naive-missing over-answer rate showed no change.

### 3.2 Kill Condition Assessment

The kill condition is not triggered: partial-refusal over-answer reduction (33.3 pp) exceeds the 10 pp minimum, and full-evidence recall change (+25.0 pp) does not constitute a loss exceeding 5 pp.

## 4. Limitations

1. **Small evaluation set.** The held-out evaluation comprises only 72 examples. Metric estimates on this sample have high variance; the observed 0.000 partial-refusal rate and 1.000 recall rate for the adapter may not generalize to larger or differently distributed evaluation sets.

2. **Single seed.** Only one random seed was used for training. The results may be sensitive to initialization and data shuffling.

3. **Single model.** The ablation was conducted only on `microsoft/Phi-4-mini-instruct`. Whether the finding transfers to other model families, sizes, or architectures is unknown.

4. **Single training configuration.** Only rank-8 LoRA with 80 microsteps was tested. The sensitivity of the result to rank, learning rate, number of steps, and data composition has not been characterized.

5. **Potential overfitting signal.** The complete elimination of partial-refusal over-answer (0.333 → 0.000) and the perfect recall score (0.750 → 1.000) on a 72-example held-out set could indicate overfitting to the evaluation distribution rather than genuine behavioral change. A larger, independently sourced evaluation set would be needed to distinguish these explanations.

6. **No external benchmark validation.** Results are limited to the project-local human-graded hardening set. Performance on standard benchmarks (e.g., MMLU, TruthfulQA) was not measured, so potential regressions on general capabilities are uncharacterized.

7. **Corpus provenance.** The training and evaluation data derive from a parent project's human-graded set. The annotation criteria, inter-annotator agreement, and domain coverage of that corpus are not documented in the available artifacts.

8. **Adapter eval throughput anomaly.** The observed adapter eval throughput being approximately 3× the base eval throughput is unexplained and warrants investigation in follow-up work.

## 6. Conclusion

The current project artifacts support the finding that, in the tested setting, audited partial-evidence hard-negative LoRA SFT can reduce partial-refusal over-answer behavior on `microsoft/Phi-4-mini-instruct` while preserving or improving full-evidence answer recall. The observed 33.3 percentage-point reduction in partial-refusal over-answer and 25.0 percentage-point improvement in full-evidence recall on the 72-example held-out set exceeded the pre-registered kill condition thresholds.

However, the small evaluation sample, single seed, single model, and single training configuration substantially limit the robustness of this finding. The complete elimination of partial-refusal over-answer and the perfect recall score on the held-out set may reflect the small sample size rather than a generalizable behavioral shift. The project decision recommends archiving this pilot result and, if further budget is allocated, replicating the ablation with multiple seeds and a larger held-out split before treating it as robust.
