## Abstract

We report empirical evidence on the reliability of landmark-based prefiltering versus model-guarded full-chunk retrieval in retrieval-augmented generation (RAG) pipelines. Using a Qwen2.5-7B Q4_K_M model-backed benchmark (8 scenarios per condition) and a local SQLite FTS5 RAG backend, we observe that landmark-only accuracy degrades substantially when synthetic examples contain decoy or mixed landmarks: from 0.8125 on naive examples to 0.5000 on reliability-adversarial examples in the model-backed setting, and from 0.812 to 0.616 in the local RAG setting. Model-guarded all-chunks retrieval maintains higher accuracy under both conditions (1.0000 naive, 0.84375 reliability in the model-backed setting; 1.000 reliability in the local RAG setting). These results support the hypothesis that landmark-only prefiltering is fragile under adversarial landmark conditions, but the evidence is bounded by small sample sizes, a single model, and the absence of a concrete production KV-offload serving stack. This routing branch was finalized positive on the strength of copied parent evidence, without executing new production-scale benchmarks.

## 1. Introduction

Retrieval-augmented generation (RAG) systems commonly use landmark-based prefiltering to reduce the number of chunks passed to the model, trading recall for inference efficiency. A key reliability concern is whether such landmarks remain trustworthy when documents contain decoy landmarks—superficially relevant markers that misdirect retrieval—or mixed landmarks that create ambiguity.

This report documents evidence from two complementary experimental backends:

1. A **model-backed benchmark** using Qwen2.5-7B (Q4_K_M quantization) served via llama.cpp, comparing `model_landmark_only` and `model_guarded_all_chunks` retrieval strategies across naive and reliability-adversarial synthetic scenarios.
2. A **local SQLite FTS5 RAG backend** comparing landmark-prefilter and guarded retrieval strategies on the same scenario classes.

The project under which this evidence was collected ("Production RAG KV-Offload Landmark Reliability Scale-Up") was an autogenerated routing branch intended to explore production-scale KV-offload implications. It contained no executable production RAG serving stack, concurrency plan, or KV-offload backend definition. The branch was finalized positive based on copied parent evidence, and no new production-scale benchmarks were executed. The findings reported here are therefore limited to the parent-project model-backed and local-RAG prototype evidence.

## 2. Method

### 2.1 Experimental Design

Two retrieval strategies were compared across two scenario classes:

**Retrieval strategies:**
- **Landmark-only prefiltering** (`model_landmark_only` / `landmark-prefilter`): Only chunks whose landmarks match the query are retrieved and presented to the model.
- **Model-guarded all-chunks** (`model_guarded_all_chunks` / `guarded backend`): All chunks are presented to the model, which guards against landmark-based misdirection by evaluating full chunk content.

**Scenario classes:**
- **Naive set**: Synthetic examples where landmarks are consistently helpful and non-deceptive.
- **Reliability set**: Synthetic examples containing decoy or mixed landmarks designed to mislead landmark-only retrieval.

### 2.2 Model-Backed Benchmark

- **Model**: Qwen2.5-7B, Q4_K_M quantization, served via llama.cpp.
- **Replication**: 8 scenarios per condition (`per_scenario=8`).
- **Metrics**: Accuracy (correct answer selection), throughput (samples/s), API token throughput (tokens/s), p95 latency (seconds), GPU utilization (%), server memory (RSS/PSS in GB), system memory available (GB).

### 2.3 Local RAG Backend

- **Backend**: SQLite FTS5 full-text search.
- **Metrics**: Accuracy on naive and reliability scenario sets.

### 2.4 Evidence Provenance

The model-backed and local RAG evidence was generated in a parent project (`source-record-redacted`) and copied into this routing branch under `evidence/parent_model_backed/` and `evidence/parent_local_rag/`. No new benchmarks were executed in this branch.

## 3. Results

### 3.1 Model-Backed Benchmark: Accuracy

| Scenario Class | `model_guarded_all_chunks` | `model_landmark_only` | Guarded-over-Landmark Uplift |
|---|---|---|---|
| Reliability | 0.84375 | 0.50000 | +0.34375 |
| Naive | 1.00000 | 0.81250 | +0.18750 |

Landmark-only accuracy drops from 0.81250 on naive examples to 0.50000 on reliability examples, a decline of 0.31250. This exposes the fragility of landmark-only prefiltering when landmarks are decoy-like or mixed. Model-guarded all-chunks retrieval also shows a decline (1.00000 to 0.84375, a drop of 0.15625), but retains substantially higher accuracy under adversarial conditions.

|---|---|---|---|

The local RAG backend shows the same directional pattern: landmark-prefilter accuracy on the reliability set (0.616) is substantially lower than on the naive set (0.812), confirming that adversarial landmarks degrade prefilter performance. ## 4. Limitations

1. **Small sample size.** The model-backed benchmark uses only 8 scenarios per condition. Confidence intervals on accuracy differences are wide; the observed uplifts are suggestive but not statistically decisive at conventional thresholds with this sample count.

2. **Single model and quantization.** Results are reported for Qwen2.5-7B Q4_K_M only. Generalization to other models, quantization levels, or serving backends is not established.

3. **Synthetic scenarios.** The naive and reliability scenario sets are synthetic. Real-world landmark failure modes may differ in distribution and severity.

4. **No production KV-offload stack.** This routing branch contained no executable production RAG serving stack, KV-offload backend, concurrency schedule, or production telemetry plan. No new benchmarks were executed. All evidence was copied from a parent project.

5. **No concurrency or scale data.** The throughput and latency figures reflect single-session prototype measurements. Production behavior under concurrent load, with KV-offload to CPU/disk, is not characterized.

6. **Local RAG backend differences.** The SQLite FTS5 backend and the model-backed benchmark use different retrieval and evaluation pipelines. Direct numerical comparison between them is not meaningful; only the directional pattern (landmark fragility under adversarial conditions) is consistent.

7. **Routing branch status.** This project was an autogenerated optional successor branch. The decision to finalize positive was based on copied parent evidence, not on new experimental work within this branch.

8. **No external replication.** Results have not been independently replicated on different hardware, by different operators, or with different random seeds beyond what is captured in the parent artifacts.

## 6. Conclusion

The evidence from the model-backed Qwen2.5-7B benchmark and the local SQLite FTS5 RAG backend consistently supports the hypothesis that landmark-only prefiltering is fragile under adversarial landmark conditions. In both backends, landmark-only accuracy degrades substantially when scenarios contain decoy or mixed landmarks, while model-guarded all-chunks retrieval maintains higher accuracy. The guarded-over-landmark uplift is +0.34375 in the model-backed reliability set and +0.384 in the local RAG reliability set.

However, these findings are bounded by small sample sizes, a single model, synthetic scenarios, and the absence of production-scale KV-offload or concurrency characterization. The routing branch under which this report was generated did not execute new benchmarks; it consolidated parent-project evidence and finalized positive. A concrete production RAG KV-offload scale-up—specifying a serving stack, concurrency plan, and telemetry target—remains necessary before these prototype results can be generalized to production deployments.
