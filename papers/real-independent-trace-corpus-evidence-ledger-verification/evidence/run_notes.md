## Hypothesis

An externally anchored evidence ledger over real agent trace logs should detect post-hoc trace tampering that passes raw JSON parsing, schema-only validation, and in some cases hash-chain-only validation. The direct target metric is tamper detection rate on real stored Codex/Enoch JSONL traces, with clean acceptance as the false-positive control.

## Implementation

Added `scripts/ledger_verify_real_traces.py`.

The script discovers real stored Codex/Enoch traces under `*/.enoch/logs/*.codex.jsonl`, parses each JSONL event, canonicalizes the event plus source path and line index, and builds a SHA-256 hash chain. Each trace also receives an anchored manifest containing `trace_id`, `source_path`, `source_sha256`, `event_count`, and final `root_hash`.

Policies:

- `raw_json_accept`: accepts any parseable JSON records.
- `schema_only`: checks ledger record fields and hash string shape, but not hash correctness.
- `hash_chain_only`: checks per-record hash and previous-hash links, but has no external expected root/count.
- `anchored_root`: checks schema, chain, event count, and expected anchored root.
- `anchored_root_source_bound`: additionally checks all records match the anchored source path.

Attacks:

- `mutate_command_output`
- `mutate_exit_code`
- `delete_event`
- `reorder_adjacent`
- `append_unanchored`
- `truncate_tail`
- `recompute_after_mutation`

Fixed seeds: `101, 202, 303, 404, 505`.

## Interpretation

The real-corpus result supports the scoped mechanism: for independently produced stored agent traces, an externally anchored root plus event count converts trace-integrity checking from best-effort local consistency into reliable tamper detection. Raw JSON and schema-only baselines provide no tamper detection. Hash-chain-only is a partial control: it catches local content mutations and most middle deletions, but it cannot detect a valid appended record, tail truncation, or a fully recomputed chain without an external anchor.

This is direct evidence on real trace logs, not synthetic traces. It is not end-to-end evidence that a live agent's final answer is protected from poisoned evidence, and it does not test private production logs, human adjudication, or cryptographic signing/key management beyond a local anchored manifest. The result is useful no-paper evidence, not publication readiness.
