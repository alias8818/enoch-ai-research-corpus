## Abstract

Long-context inference systems that selectively rehydrate cached context segments can reduce resident memory, but routing decisions—whether to skip, sketch-rehydrate, fall back to full context, or abstain—must be made cheaply before full context is loaded. We train a compact 72-parameter multiclass perceptron student to approximate a cost-aware guide teacher's routing policy using only cheap task and sketch features (anchor recall, sketch entropy, ambiguity, compression loss, budget pressure, answer criticality, locality, and unanswerable probability). On a synthetic policy-isolation benchmark spanning six task types (kv_recall, section_qa, reversible_compression, structured_extract, code_nav, unanswerable), the student matches guide-teacher labels at 98.01% mean accuracy across five random seeds (range 97.28%–98.50%). The student achieves mean utility 0.8718 versus the guide-teacher oracle's 0.8771 (mean gap 0.0052), gains 0.3723 mean utility over a static hand-coded heuristic baseline, and reduces normalized cost by 27.17% compared to a full-context-always fallback (30.59 vs. 42.00 mean normalized cost). These results are limited to synthetic policy isolation with hand-modeled utilities; no live LLM, real inference trace, or real KV-cache integration was evaluated. The findings support the feasibility of cheap learned routing for cost-aware rehydration but require validation on real rehydration traces with downstream answer-quality metrics before any deployment claim.

## Introduction

Long-context language model inference faces a fundamental tension: maintaining full context in resident memory is expensive, yet discarding context risks degrading answer quality. Context rehydration—the selective reloading of cached context segments into active memory—offers a potential middle path, but only if routing decisions are both accurate and cheap. These decisions must be made before full context is loaded, using only lightweight features available at routing time.

Prior local experiments on section-sketch rehydration (project `source-record-redacted`) found that exact section anchors can preserve quality at much lower resident memory, while lossy topic-only sketches fail. Separately, prior local experiments with context-rehydration students (project `source-record-redacted`) found that task-specific student models can outperform undistilled heuristics on synthetic sketch-memory tasks. These findings motivate a central question: can a compact student model learn the routing policy of a cost-aware rehydration guide—deciding among actions such as skip, sketch-rehydrate, teacher fallback, and abstain—using only cheap features available before full context is loaded?

This paper addresses that question in a controlled, synthetic setting. We define a cost-aware guide teacher that labels the best routing action by expected answer utility minus action cost, then train a small multiclass perceptron student on the resulting labels using only inexpensive features. We evaluate against three baselines: a static hand-coded heuristic, a no-rehydrate policy, and a full-context-always fallback. The goal is not to demonstrate absolute quality dominance over unlimited full context—full-context-always will naturally achieve higher raw utility when cost is ignored—but rather to show that cost-quality tradeoffs can be navigated effectively by a learned router at very low overhead.

## Method

### Benchmark Design

The benchmark (`src/rehydration_guide_student.py`) generates synthetic context-rehydration routing cases across six task types:

- **kv_recall**: Key-value lookup requiring precise anchor access.
- **section_qa**: Section-level question answering.
- **reversible_compression**: Tasks where compressed context can be losslessly decompressed.
- **structured_extract**: Extraction from structured data.
- **code_nav**: Code navigation requiring locality-aware access.
- **unanswerable**: Cases where no available context supports an answer.

Each case is characterized by eight cheap features computable without loading full context: anchor recall, sketch entropy, ambiguity, compression loss, budget pressure, answer criticality, locality, and unanswerable probability.

### Actions

The routing policy selects among four actions:

- **skip**: Do not rehydrate; proceed with current context.
- **sketch_rehydrate**: Load a compressed sketch of the relevant context segment.
- **teacher_fallback**: Fall back to full context (highest cost, highest potential utility).
- **abstain**: Decline to answer.

### Guide Teacher

A cost-aware guide teacher labels each case with the action maximizing expected answer utility minus action cost. The teacher's utility model is hand-constructed from prior local evidence about section-anchor effectiveness and sketch failure modes. This is a modeling assumption, not ground truth from real inference traces. If the teacher's utility model is miscalibrated, the student inherits those errors.

### Student Model

The student is a 72-parameter multiclass perceptron trained on the eight cheap features to predict the guide teacher's action label. Training uses standard cross-entropy loss. The model is intentionally small to ensure low inference overhead.

### Baselines

1. **Static heuristic**: A hand-coded routing rule using fixed thresholds on features.
2. **No-rehydrate policy**: Always skips rehydration.
3. **Full-context-always fallback**: Always selects `teacher_fallback`, maximizing potential utility at maximum cost.

### Evaluation Protocol

A corrected 5-seed calibrated sweep was run using `src/run_guide_student_sweep.py`. Each seed generates fresh synthetic cases, trains the student, and evaluates on held-out cases. Metrics include label accuracy against the guide teacher, mean utility, normalized cost, and per-seed consistency. An initial smoke test and sweep were corrected to include the full-context baseline; all reported results use the corrected (v2) runs. Pre-declared success criteria were: label accuracy ≥ 0.80, positive utility gain over the static heuristic, and ≥ 25% cost saving versus full-context-always fallback.

## Results

### Label Accuracy

Across five random seeds, the student matches guide-teacher labels at mean accuracy 0.9801 (range 0.9728–0.9850). This indicates the cheap feature set is sufficient for the perceptron to closely approximate the teacher's routing policy on the synthetic distribution. The narrow range across seeds suggests the approximation is stable under the synthetic distribution, though stability under distribution shift remains untested.

### Utility

| Policy | Mean Utility |
|---|---|
| Guide-teacher oracle | 0.8771 |
| Student | 0.8718 |
| Static heuristic (derived) | ~0.4995 |

The mean utility gap between student and guide-teacher oracle is 0.0052. The student gains 0.3723 mean utility over the static heuristic baseline. The static heuristic's absolute utility (~0.4995) is derived by subtracting the reported gain (0.3723) from the student utility (0.8718); it was not reported as a standalone metric and should be treated with caution.

The full-context-always policy achieves higher raw utility than the student when cost is not penalized; the student's advantage is in cost-quality tradeoff, not absolute quality. This is an inherent property of cost-aware routing: the student sometimes selects cheaper actions that sacrifice marginal utility, and this is the intended behavior.

### Cost

| Policy | Mean Normalized Cost |
|---|---|
| Student | 30.5884 |
| Full-context-always | 42.0000 |

The student achieves 27.17% mean cost saving versus full-context-always. This is the primary cost-quality claim: the student routes most cases away from the most expensive action while preserving most of the guide teacher's utility.

### Overhead

The student model comprises 72 parameters and processes 1,000 examples in 3.178 ms in the Python benchmark loop. This overhead is negligible relative to typical LLM inference latency. However, this measurement is a toy Python-loop benchmark, not representative of production inference-stack integration; real overhead depends on feature extraction cost and integration architecture, neither of which was evaluated.

### Per-Seed Consistency

Results are consistent across seeds. The minimum label accuracy (0.9728) and the narrow range of utility gaps suggest the student's approximation is stable under the synthetic distribution. Whether this consistency holds under distribution shift or different task mixtures is unknown.

## Limitations

1. **Synthetic policy isolation only.** No live LLM, real inference trace, or real KV-cache integration was evaluated. The benchmark models routing decisions in isolation; it does not measure downstream answer quality from an actual language model. This is the most significant limitation.

2. **Hand-modeled guide utilities.** The guide teacher's utility function is constructed from prior local evidence about anchor and sketch effectiveness, not from real rehydration traces. Real task distributions, cost structures, and action outcomes could shift thresholds and change the student's learned policy. The 98% label accuracy measures agreement with a potentially miscalibrated teacher, not correctness against real outcomes.

3. **Full-context-always has higher raw utility when cost is ignored.** The result supports cost-aware routing, not absolute quality dominance over unlimited full context. Any deployment claim must account for this asymmetry: the student is trading marginal quality for cost savings, and this tradeoff is only justified when cost matters.

4. **No downstream answer-quality evaluation.** Label accuracy against the guide teacher measures policy agreement, not whether the routed actions produce good answers from a real model. A student that agrees with a miscalibrated teacher inherits the teacher's errors.

5. **Feature availability assumption.** The eight cheap features (anchor recall, sketch entropy, ambiguity, compression loss, budget pressure, answer criticality, locality, unanswerable probability) are assumed to be available at routing time. In a real system, some features may require estimation or may not be available before context loading, which could degrade student accuracy.

6. **Toy latency measurement.** The 3.178 ms per 1k examples was measured in a Python loop, not in an inference-stack integration. Production latency characteristics, including feature extraction overhead, may differ substantially.

7. **Static heuristic utility derived, not directly reported.** The static heuristic's absolute utility (~0.4995) is derived by subtracting the reported gain (0.3723) from the student utility (0.8718), not reported as a standalone metric. This derivation should be treated with caution.

8. **Claim audit incomplete.** The claim ledger for this draft contains no completed claim-audit entries and flags that a human claim audit is required. All numerical results in this paper should be read as unverified prototype evidence pending independent audit.

## Conclusion

A 72-parameter multiclass perceptron student can closely approximate a cost-aware context-rehydration guide's routing policy on a synthetic benchmark, achieving 98.01% mean label accuracy, a utility gap of 0.0052 versus the guide-teacher oracle, a 0.3723 utility gain over a static heuristic, and 27.17% cost savings versus full-context-always fallback. These results demonstrate the feasibility of cheap learned routing for cost-aware rehydration in a controlled, synthetic setting and meet all three pre-declared success criteria.

However, the evidence is limited to synthetic policy isolation with hand-modeled utilities. The 98% accuracy measures agreement with a teacher whose own calibration is unvalidated against real outcomes. Before any deployment claim, the approach requires: (1) validation on real offline rehydration traces with action, cost, and outcome labels; (2) integration into a minimal long-context inference wrapper with memory telemetry; and (3) evaluation of downstream answer quality and fallback calibration rather than synthetic label agreement alone. The results are promising but medium-confidence; the gap between synthetic policy isolation and real inference-stack integration remains the central risk.
