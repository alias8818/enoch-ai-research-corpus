## Abstract

Long-context transformer serving incurs growing KV-cache memory and quadratic prefill attention costs as prompt length increases. This paper evaluates a serving-boundary context compression strategy in which a compact summary of prior context replaces the original prompt prefix at the boundary between consecutive inference requests, retaining only the summary plus a recent token tail in the KV cache. We report analytical KV-cache projections across four compression scenarios (8k→1.5k, 16k→1.5k, 32k→3k, 64k→5k retained tokens) under an 8B-parameter grouped-query attention (GQA) geometry, finding KV-cache reductions of 81.25%–92.19% and prefill-work proxy reductions of 96.48%–99.39%. We supplement these projections with single-thread NumPy synthetic attention microbenchmarks on an aarch64 host, which confirm that decode-side attention cost scales with retained context length as expected (68× latency increase from 512 to 16,384 tokens). These results establish that boundary compression yields substantial serving-side resource savings in principle. However, no semantic quality evaluation was performed, and the microbenchmark uses CPU synthetic attention rather than a production CUDA inference server. The central open question—whether compressed summaries preserve task accuracy—remains unresolved.

## 1. Introduction

Transformer-based language model serving faces two pressure points as context windows grow. First, KV-cache memory scales linearly with the number of retained prompt tokens; for an 8B-class GQA model, a single 64k-token context occupies approximately 8 GiB of KV-cache memory. Second, prefill attention computation exhibits quadratic dependence on prompt length, making time-to-first-token (TTFT) a growing concern for long-context workloads.

A natural strategy is to compress the prompt at the *serving boundary*—the point between consecutive inference requests—by replacing the full prior context with a compact summary and retaining only this summary plus a recent token tail. If the summary preserves sufficient semantic content, the KV-cache footprint and prefill work for subsequent requests can be substantially reduced.

This paper does not address whether such summaries preserve answer quality. Instead, it quantifies the serving-side resource savings that boundary compression would provide *if* a sufficiently faithful summary were available. We combine an analytical KV-cache model with synthetic CPU microbenchmarks to characterize the magnitude of potential gains and the cost direction of retained-context length on decode-side attention. The results are intended to establish whether the serving-side mechanics are viable enough to justify the more difficult quality evaluation.

## 2. Method

### 2.1 Analytical KV-Cache Model

KV-cache memory per request is modeled as:

$$M_{\text{KV}} = n_{\text{tokens}} \times n_{\text{layers}} \times n_{\text{KV heads}} \times d_{\text{head}} \times 2 \times b$$

where the factor of 2 accounts for K and V projections and $b$ is the bytes per element (2 for fp16). For the reference 8B-like GQA geometry used throughout (32 layers, 8 KV heads, 128 head dimension, fp16), this yields 131,072 bytes per token.

Prefill work is proxied as proportional to $n_{\text{tokens}}^2$ for full self-attention over the prompt, reflecting the quadratic cost of the attention computation. The prefill-work proxy reduction under compression is computed as:

$$1 - \frac{n_{\text{retained}}^2}{n_{\text{original}}^2}$$

Four compression scenarios are evaluated, representing progressively longer original contexts with varying summary lengths:

| Scenario | Original tokens | Retained (summary + tail) tokens |
|---|---:|---:|
| 8k_chat_to_1k_summary | 8,192 | 1,536 |
| 16k_agent_trace_to_1k_summary | 16,384 | 1,536 |
| 32k_agent_trace_to_2k_summary | 32,768 | 3,072 |
| 64k_long_case_to_4k_summary | 65,536 | 5,120 |

These scenarios span chat, agent trace, and long-case use cases with compression ratios from approximately 5:1 to 13:1. The retained token counts (1,536–5,120) represent plausible summary-plus-tail lengths; they are not derived from any particular summarization model.

### 2.2 Synthetic Attention Microbenchmark

To confirm that retained-context length is a measurable cost driver on the decode path, we implemented a single-token synthetic attention benchmark using NumPy on CPU. The benchmark constructs K and V arrays of dimension `kv_dim=1024` for a given context length, then performs a single attention step (dot-product scoring, softmax, value aggregation) for one query token against the full context. This isolates the context-length-dependent portion of attention from all other inference costs.

The benchmark was executed with `OMP_NUM_THREADS=1` and `OPENBLAS_NUM_THREADS=1` to reduce thread-scheduling variance. Context lengths tested were 512, 1024, 2048, 4096, 8192, and 16384 tokens, with 11 repetitions each. Median timing is reported.

This is a toy-scale calibration on CPU, not a production inference measurement. It is included solely to confirm the expected cost direction—that reducing retained context length reduces decode-side attention latency—and to provide a rough sense of scaling behavior.

## 3. Results

### 3.1 Analytical KV-Cache and Prefill Projections

Table 1 reports the analytical KV-cache savings and prefill-work proxy reductions across all four scenarios under the reference GQA geometry.

**Table 1.** KV-cache and prefill-work proxy reduction under serving-boundary compression (8B-like GQA, fp16).

| Scenario | Original tokens | Retained tokens | KV original (GiB) | KV compressed (GiB) | KV saved (GiB) | KV reduction | Prefill proxy reduction |
|---|---:|---:|---:|---:|---:|---:|---:|
| 8k→1.5k | 8,192 | 1,536 | 1.000 | 0.188 | 0.812 | 81.25% | 96.48% |
| 16k→1.5k | 16,384 | 1,536 | 2.000 | 0.188 | 1.812 | 90.63% | 99.12% |
| 32k→3k | 32,768 | 3,072 | 4.000 | 0.375 | 3.625 | 90.63% | 99.12% |
| 64k→5k | 65,536 | 5,120 | 8.000 | 0.625 | 7.375 | 92.19% | 99.39% |

KV-cache reduction ranges from 81.25% (8k→1.5k) to 92.19% (64k→5k). Per-request memory savings range from 0.81 GiB to 7.38 GiB. Prefill-work proxy reduction is larger in all cases (96.48%–99.39%) due to the quadratic scaling of full-attention prefill.

These are analytical projections based on transparent model geometry assumptions, not measurements from a live inference server. They represent upper bounds on resource savings, conditional on the assumption that a summary of the stated length can be produced and is sufficiently faithful.

### 3.2 Synthetic Attention Microbenchmark

Table 2 reports median single-token attention latency and corresponding synthetic K+V array size across tested context lengths.

**Table 2.** Single-thread NumPy synthetic one-token attention (kv_dim=1024, 11 reps, median reported).

| Context tokens | Median latency (ms) | Synthetic K+V array (MiB) |
|---:|---:|---:|
| 512 | 0.0542 | 4.0 |
| 1,024 | 0.1176 | 8.0 |
| 2,048 | 0.2678 | 16.0 |
| 4,096 | 0.6828 | 32.0 |

Median latency increased by a factor of 68.0 from 512 to 16,384 retained tokens. The growth is super-linear but sub-quadratic, consistent with the mix of memory-bound array traversal and compute-bound dot products in the NumPy implementation. This confirms the expected cost direction: reducing retained context length reduces decode-side attention latency. However, the exact scaling will differ on production CUDA hardware with Flash Attention, tensor cores, and GPU memory hierarchy effects. The absolute latency numbers should not be compared to production decode costs.

## 4. Limitations

1. **No semantic quality evaluation.** The central claim of serving-boundary compression—that a summary can replace full context—depends critically on whether the compressed representation preserves task-relevant information. No accuracy, perplexity, or task-performance measurement was conducted. The reported resource savings are upper bounds that assume perfect summary fidelity. A summary that loses critical information may produce resource savings that are real but practically unusable.

2. **Synthetic CPU microbenchmark, not production inference.** Timing measurements use NumPy CPU synthetic attention with `kv_dim=1024`, not a CUDA-based inference server with Flash Attention, KV-cache paging, or tensor parallelism. The absolute latency numbers are not representative of production decode costs, and the scaling behavior will differ on GPU hardware. This microbenchmark serves only as a toy-scale calibration confirming the expected cost direction.

3. **Analytical KV geometry assumptions.** The 8B-like GQA geometry (32 layers, 8 KV heads, 128 head dimension, fp16) is a transparent benchmark assumption, not derived from live inspection of a deployed model. Other model configurations (e.g., MQA, larger head dimensions, fp8 quantization) will yield different absolute savings but the same directional conclusion.

4. **Prefill proxy is a simplification.** The $n^2$ prefill-work proxy ignores kernel-level optimizations (Flash Attention, tensor cores), prefix caching, and chunked prefill. Actual TTFT reduction will depend on implementation details not modeled here.

5. **Single host, single configuration.** All measurements were collected on one aarch64 host. No multi-GPU, multi-host, or alternative-architecture results are available.

6. **No concurrent-load evaluation.** The benchmarks measure single-request cost in isolation. Under concurrent request load, KV-cache savings translate into admission control and batching benefits that were not quantified.

7. **Retained token counts are assumed, not optimized.** The summary lengths (1,536–5,120 tokens) are plausible but arbitrary. The optimal compression ratio for a given task is unknown and likely task-dependent.

## 6. Conclusion

Serving-boundary context compression—replacing a long prompt prefix with a compact summary at the boundary between inference requests—offers substantial theoretical reductions in KV-cache memory (81%–92%) and prefill computation (96%–99%) for 8B-class GQA models across context lengths from 8k to 64k tokens. Synthetic CPU microbenchmarks confirm that decode-side attention cost scales with retained context length (68× latency increase from 512 to 16,384 tokens), supporting the directional claim that compression reduces per-request serving cost.

However, these results address only the serving-side mechanics. The decisive open question is whether compressed summaries preserve sufficient task accuracy to make the resource savings practically usable. The analytical and microbenchmark evidence presented here justifies—but does not substitute for—a paired quality-and-serving benchmark on a real model, measuring answer quality, time-to-first-token, decode latency, and live KV telemetry under full-context versus summary-plus-tail prompts. Until such an experiment is conducted, the resource savings reported here should be treated as upper bounds conditional on summary fidelity.
