## Abstract

We investigate whether a Sparse-Value Late Materializer (SVLM) can preserve answer-critical old-context state under tighter fast-memory budgets better than recency-only eviction for long-context transformer inference. Three policies—recency-only eviction, top-k sparse value hints, and late dense-value fetch—are compared against exact full attention across three synthetic scenarios: retrieval-sparse, diffuse summarization, and adversarial dense. In a CPU/Numpy mechanism benchmark at 16,384 tokens with 2,048 recent tokens (12.5%), late dense-value fetch achieves near-exact reconstruction on retrieval tasks (mean relative L2 error ≈ 0.0018) while reducing fast-memory occupancy by ~43%. However, it catastrophically fails on diffuse summarization (relative L2 error ≈ 13.6–15.2) unless the fetch count approaches the full cold set. Pure top-k sparse value hints improve over recency for retrieval-sparse attention (relative L2 ≈ 0.20–0.24) but remain inadequate for dense or adversarial value distributions (relative L2 ≈ 0.70–0.81). Recency-only eviction yields ~1.0 relative L2 error on retrieval and ~3.9 on diffuse summarization despite 87.5% memory reduction. These results support conditional viability for a hybrid approach: late-fetch for concentrated attention heads, with full retention or quantization for diffuse heads. Pure sparse value hints alone are insufficient as a primary mechanism. All results derive from synthetic attention tensors, not model-integrated task evaluation.

## 1. Introduction

Long-context transformer inference faces a growing memory bottleneck at the key-value (KV) cache. As context lengths increase, the memory required to store dense KV pairs for all prior tokens dominates GPU fast-memory budgets. Prior work has proposed attention-guided eviction, layer-condensed caching, and dictionary-based compression to reduce this cost, but practical deployment requires balancing memory savings against accuracy loss across diverse attention patterns—not only aggregate compression ratios.

A natural question arises: can we avoid storing dense values for cold (non-recent) tokens entirely, instead keeping only compact hints and restoring dense values on demand when attention signals access? This idea decomposes into two sub-mechanisms:

1. **Sparse value hints**: retain all keys but store only the top-k value coordinates for cold tokens, reconstructing or approximating the rest.
2. **Late dense fetch**: retain all keys and recent dense values in fast memory; offload cold dense values to slower memory and fetch only the top-m values indicated by attention scores at query time.

If either mechanism preserves answer-critical state under tight memory budgets, it opens a path to substantial fast-memory reduction for long-context inference. However, the viability of each mechanism depends critically on the structure of attention: retrieval tasks concentrate attention on few cold tokens, while summarization and aggregation tasks distribute attention broadly.

This paper reports a synthetic mechanism benchmark that isolates the attention/KV interaction from any particular model. We compare three policies against exact full attention across three scenarios designed to stress different attention distributions. Our goal is to determine whether the SVLM idea warrants a model-integrated next step, and if so, in what form.

## 2. Method

### 2.1 Policies

We compare three KV-cache management policies against the baseline of exact full attention:

**Recency-only.** Retain only the most recent $r$ tokens' full key-value pairs in fast memory. All older tokens are evicted. This is the simplest eviction policy and serves as a lower-bound baseline for memory savings but an upper-bound baseline for error.

**Sparse value.** Retain all keys in fast memory. For recent tokens (most recent $r$), retain full values. For cold tokens (tokens older than $r$), retain only the top-$k$ value coordinates by magnitude, zeroing all others. This tests whether a small number of dominant value coordinates can preserve answer-critical information.

**Late fetch.** Retain all keys and recent full values in fast memory. Cold dense values are conceptually offloaded to slower memory. At query time, attention scores over keys determine which cold tokens are accessed; the top-$m$ cold dense values are fetched back into fast memory. This models an offload/restore path rather than sparse reconstruction.

### 2.2 Synthetic Scenarios

We construct three scenarios to exercise different attention distributions:

- **retrieval_sparse**: Old answer-critical needles dominate attention. A small number of cold tokens receive high attention weights; the rest receive near-zero weight. This models needle-in-a-haystack retrieval.
- **diffuse_summary**: The answer depends on broad aggregation over many old tokens. Attention weights are approximately uniform over the cold span. This models summarization or document-level extraction.
- **adversarial_dense**: Old answer-critical tokens are retrieved (concentrated attention), but their values are dense—i.e., important information is spread across many value coordinates rather than concentrated in a few top-k components. This tests whether sparse value hints can handle non-sparse value distributions even when attention is favorable.

### 2.3 Error Metric

We measure reconstruction quality as relative L2 error:

$$\text{rel\_l2} = \frac{\|V_{\text{approx}} - V_{\text{exact}}\|_2}{\|V_{\text{exact}}\|_2}$$

where $V_{\text{approx}}$ is the value matrix produced by the policy and $V_{\text{exact}}$ is the full-attention value matrix. A relative L2 of 0.0 indicates exact reconstruction; 1.0 indicates error equal in magnitude to the signal.

### 2.4 Memory Accounting

Memory is accounted in units of fp16/bf16 elements (2 bytes per element), consistent with production KV-cache storage, even though computation uses fp32 in Numpy. Fast-memory occupancy includes all keys, all recent values, and (for sparse value) the stored sparse coordinates. For late fetch, fast-memory occupancy includes all keys and recent values but excludes offloaded cold dense values; the per-query fetch cost is reported separately in bytes.

### 2.5 Configuration

The representative main-run configuration is:

| Parameter | Value |
|---|---|
| Total tokens ($n$) | 16,384 |
| Recent tokens ($r$) | 2,048 (12.5%) |
| Sparse value $k$ | 32 |
| Late fetch $m$ | 64 |
| Value dtype accounting | fp16/bf16 (2 bytes) |

The main run sweeps multiple configurations (54 rows total); the representative configuration above is used for primary reporting.

### 2.6 Execution Environment

All experiments run on a GB10 host using CPU/Numpy execution (no GPU computation). Python 3.12.3. No swap was configured or observed. The main run completed in 6.34 seconds wall time with 186,836 kB maximum RSS. This is a toy simulation / synthetic mechanism benchmark: it isolates the KV-policy interaction using constructed attention tensors on CPU, and does not involve model inference, CUDA kernels, or GPU memory hierarchies.

### 3.3 Analysis by Policy

**Recency-only eviction** achieves the largest memory reduction (87.5% in the representative configuration) but at severe accuracy cost. On retrieval and adversarial scenarios, relative L2 error is approximately 1.0—meaning the error is as large as the signal itself. On diffuse summarization, the error is approximately 3.9, indicating that the evicted cold values carry most of the answer-critical information. Recency-only eviction is a weak baseline for any task that depends on old-context state.

**Sparse value hints** improve over recency when the value distribution is sparse (retrieval_sparse: rel-L2 ≈ 0.20–0.24). However, this improvement is not sufficient for high-fidelity reconstruction, and it degrades severely when values are dense. On adversarial_dense and diffuse_summary, relative L2 error is 0.71–0.81, indicating that the top-32 value coordinates retain too little information. This is a **negative result**: pure top-k sparse value hints cannot robustly reconstruct dense answer-critical values and should not be pursued as the sole mechanism without a learned dictionary or reconstructor.

**Late dense-value fetch** achieves near-exact reconstruction on retrieval and adversarial scenarios (rel-L2 ≈ 0.0018) while reducing fast-memory by ~43%. The per-query fetch cost at $m{=}64$ is 32,768 bytes (64 cold values × 512 value-dim × 2 bytes, consistent with the reported configuration). However, late fetch catastrophically fails on diffuse summarization (rel-L2 ≈ 13.6–15.2) because the top-64 cold values capture only a small fraction of the uniformly-attended cold span. Making late fetch work for diffuse attention would require fetching a large fraction of the cold set, erasing the traffic and latency benefits.

### 3.4 Scenario-Dependent Viability

The results reveal a sharp dichotomy:

- **Concentrated attention** (retrieval, adversarial): Late fetch is highly effective. Sparse value hints offer moderate improvement over recency but are not competitive with late fetch.
- **Diffuse attention** (summarization): No policy that reduces cold-value storage works well. Recency eviction, sparse hints, and late fetch all fail. Full retention, quantization, or periodic refresh mechanisms are needed.

This dichotomy motivates a hybrid design with an attention-concentration gate.

## 4. Limitations

1. **Synthetic attention tensors.** All experiments use constructed attention patterns, not attention weights from a real language model. The relative L2 error on synthetic tensors does not directly predict task-level metrics (F1, exact match, perplexity) on real model outputs. Final scientific closure requires model-integrated evaluation.

2. **No model-integrated benchmark.** This work isolates the KV mechanism. It does not measure end-to-end inference throughput, latency, or task accuracy on any LLM. The memory accounting is analytical rather than measured on GPU hardware.

3. **Limited scenario coverage.** Three synthetic scenarios cannot capture the full diversity of real-world attention patterns. Real models may exhibit intermediate concentration levels, head-wise variation, and layer-wise variation not represented here.

4. **No learned components.** The sparse value policy uses magnitude-based top-k selection. A learned dictionary or reconstructor might improve sparse value reconstruction, but this was not tested. The negative result applies specifically to unlearned magnitude-based sparse hints.

5. **Single sequence length.** The representative configuration uses 16,384 tokens. Behavior may differ at shorter or longer context lengths.

6. **No latency modeling.** Late fetch assumes that offloaded cold values can be retrieved with acceptable latency. The benchmark does not model the actual cost of device-to-device transfers or their impact on inference latency.

7. **CPU-only execution.** All computation is on CPU with Numpy. No CUDA kernels, tensor cores, or GPU memory hierarchies are involved. The resource posture (6.34 s wall time, 186 MB RSS) reflects the benchmark infrastructure, not production inference cost.

## 6. Conclusion

This synthetic mechanism benchmark yields three principal findings for the Sparse-Value Late Materializer concept:

1. **Recency-only eviction is insufficient** for any task that depends on old-context state, producing relative L2 errors of ~1.0 on retrieval and ~3.9 on summarization despite 87.5% memory reduction.

2. **Pure top-k sparse value hints are not robust** as a primary mechanism. They improve over recency for sparse value distributions (rel-L2 ≈ 0.20) but fail on dense values (rel-L2 ≈ 0.71–0.81). This is a negative result for the hypothesis that sparse value hints alone can preserve answer-critical state.

3. **Late dense-value fetch is conditionally viable**: it achieves near-exact reconstruction (rel-L2 ≈ 0.0018) on concentrated-attention tasks with ~43% fast-memory reduction, but catastrophically fails on diffuse attention (rel-L2 ≈ 13.6–15.2) unless the fetch count approaches the full cold set.

These results support **conditional positive viability** for a hybrid design: an attention-concentration gate that routes concentrated heads/spans through late-fetch while retaining, quantizing, or refreshing full values for diffuse heads. The recommended next step is integration with a real long-context model, adding per-head concentration gates, and benchmarking against recency, attention-guided eviction, and KV quantization on both needle-retrieval and summarization tasks. Pure sparse value hints should not be pursued as the sole mechanism without a learned reconstruction component.
