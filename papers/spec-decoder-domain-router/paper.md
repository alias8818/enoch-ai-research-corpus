## Abstract

Speculative decoding accelerates autoregressive inference by proposing candidate token sequences from a fast draft model and verifying them against a target model. The acceptance rate depends on how closely the draft distribution matches the target distribution. We investigate whether a lightweight prompt-domain router can select a domain-specialized draft model that better approximates the target distribution than a single global draft model. Because no implementation existed at project outset, we conduct a controlled proxy experiment using n-gram language models on the 20 Newsgroups corpus, measuring acceptance overlap—the sum of element-wise minima between target and draft next-token distributions—rather than actual token acceptance in a speculative decoding loop. A TF-IDF logistic-regression router (87.7% document accuracy) selects among four domain-specific bigram draft models. On 4,000 held-out evaluation contexts, the routed draft achieves a mean acceptance overlap of 0.6177, compared to 0.5116 for the global bigram draft, a lift of +20.7%. The router captures 99.8% of the oracle domain-selection lift. A three-seed robustness sweep yields a mean lift of +20.3% (range: +20.0% to +20.7%). However, the domain-specific bigram models exhibit worse held-out cross-entropy than the global bigram model (e.g., 8.78 vs. 8.32 bits/token on the `comp` domain), indicating that the acceptance-overlap gain is specific to matching a domain-conditioned target distribution and does not constitute a general improvement in language-model quality. These results are n-gram proxy findings, not production LLM benchmarks; real speculative-decoding validation with transformer models remains necessary.

## 1. Introduction

Speculative decoding reduces inference latency by using a compact draft model to propose short token sequences, which the target model then verifies in parallel. The expected number of accepted tokens per draft step depends on the divergence between the draft and target next-token distributions: closer distributions yield fewer rejections and higher throughput.

In practice, a single global draft model must approximate the target across all input domains. When the target's behavior is domain-conditioned—for example, shifting vocabulary and topic distributions across technical, recreational, scientific, and political text—a domain-specialized draft may provide a tighter approximation for in-domain inputs. The challenge is to route each prompt to the correct domain-specific draft without introducing latency that offsets the acceptance gain.

This study asks: can a lightweight prompt-domain router select a domain-specialized draft decoder that is a better speculative-decoding proposal distribution than a single global draft decoder?

We operationalize "better proposal distribution" as higher acceptance overlap:

$$\text{acceptance\_overlap}(p_{\text{target}}, q_{\text{draft}}) = \sum_{t} \min\bigl(p_{\text{target}}(t \mid \text{context}),\; q_{\text{draft}}(t \mid \text{context})\bigr)$$

This quantity upper-bounds the per-token acceptance probability in the standard speculative-decoding accept/reject step. Higher overlap implies fewer expected target-model rejections.

Because the project directory contained only specification metadata and no implementation, we built a local proxy experiment using n-gram language models rather than transformer LLMs. This design provides a controlled, falsifiable test of the core routing mechanism while explicitly sacrificing ecological validity with respect to production speculative decoding. The experiment should be understood as a toy simulation that tests whether the routing mechanism can, under favorable conditions, produce measurable acceptance-overlap improvements—not as evidence of production-level speedup.

## 2. Method

### 2.1 Dataset and Domain Partition

We use the scikit-learn 20 Newsgroups dataset, mapping its 20 fine-grained categories into four coarse domains:

| Domain | Constituent categories |
|--------|----------------------|
| `comp` | `comp.graphics`, `comp.sys.mac.hardware`, `comp.windows.x` |
| `rec` | `rec.autos`, `rec.motorcycles`, `rec.sport.baseball` |
| `sci` | `sci.electronics`, `sci.med`, `sci.space` |
| `talk` | `talk.politics.guns`, `talk.politics.misc`, `talk.religion.misc` |

The dataset splits into 5,508 training documents and 3,641 test documents.

### 2.2 Models

**Router.** A TF-IDF vectorizer (word and phrase features) followed by a multinomial logistic regression classifier, trained on the training-split documents with their domain labels. The router predicts the domain of a given document or prompt.

**Target distribution.** An oracle per-domain trigram language model trained on one half of the training split for each domain. The target distribution is domain-conditioned: given a context known to belong to domain $d$, the target distribution is the trigram model for $d$. This is an oracle in the sense that the target's domain identity is known exactly, which is a favorable assumption for the mechanism under test.

**Draft distributions.** A global bigram language model trained on the other half of the full training split (all domains combined), and four per-domain bigram language models trained on the same half, partitioned by domain. The per-domain models see less training data than the global model—roughly one-quarter—making them weaker conventional language models but potentially better aligned with domain-specific target distributions.

### 2.3 Evaluation Protocol

For each test document, we:

1. Extract a context window and compute the target next-token distribution from the oracle domain trigram model.
2. Compute the draft next-token distribution from (a) the global bigram model, (b) the oracle domain bigram model (ground-truth domain), and (c) the router-predicted domain bigram model.
3. Measure acceptance overlap between the target and each draft distribution.
4. Report mean acceptance overlap across all evaluated contexts.

The primary comparison is the lift of the routed domain draft over the global draft. A secondary comparison is the fraction of oracle-domain lift captured by the router, which quantifies how much of the theoretically available gain the router realizes in practice.

### 2.4 Robustness Sweep

To assess sensitivity to random variation, we run three additional evaluations with seeds 1, 2, and 3, using 350 contexts per domain (1,400 total) and a vocabulary size of 3,000.

### 2.5 Resource Posture

All experiments ran CPU-only on a machine with an NVIDIA GB10 GPU (0% utilization). Swap was disabled throughout. Peak RSS was 510,848 KB. Available memory remained above 122 GB before and after the full evaluation. Wall time for the full evaluation was approximately 3 minutes and 9 seconds (188.4 s). These resource figures characterize the n-gram proxy only; they are not indicative of production LLM speculative-decoding resource requirements.

## 3. Results

### 3.1 Full Evaluation

The full evaluation used 4,000 contexts (1,000 per domain) with a vocabulary size of 5,000.

| Metric | Value |
|--------|-------|
| Router document accuracy | 0.8767 |
| Global draft mean acceptance overlap | 0.5116 |
| Oracle domain draft mean acceptance overlap | 0.6179 |
| Routed domain draft mean acceptance overlap | 0.6177 |
| Routed lift over global | +20.74% |
| Router capture of oracle lift | 99.83% |

The routed domain draft closely approximates the oracle domain draft (0.6177 vs. 0.6179), indicating that the router's domain predictions are sufficiently accurate for draft selection in this proxy setting. The near-complete capture of oracle lift (99.83%) suggests that routing errors are infrequent enough that their impact on mean acceptance overlap is negligible at this domain granularity.

### 3.2 Robustness Sweep

| Seed | Contexts | Vocab | Routed lift over global (%) | Router accuracy |
|------|----------|-------|---------------------------|-----------------|
| 1 | 1,400 | 3,000 | +20.31 | 0.8767 |
| 2 | 1,400 | 3,000 | +19.97 | 0.8767 |
| 3 | 1,400 | 3,000 | +20.67 | 0.8767 |
| **Mean** | — | — | **+20.32** | — |

The lift is stable across seeds, with a range of approximately 0.7 percentage points. Router accuracy is identical across seeds (0.8767), which is expected given that the router is trained on the same data regardless of the evaluation sampling seed.

### 3.3 Contradictory Evidence: Cross-Entropy

Despite the acceptance-overlap improvement, the per-domain bigram models are worse conventional language models than the global bigram model. Held-out bits per token from the full run:

| Domain | Global bigram (bits/token) | Domain bigram (bits/token) | Difference |
|--------|---------------------------|---------------------------|------------|
| `comp` | 8.315 | 8.783 | +0.468 |
| `rec` | 8.329 | 8.875 | +0.546 |
| `sci` | 8.193 | 8.732 | +0.539 |
| `talk` | 8.316 | 8.793 | +0.477 |

The domain bigram models exhibit 0.47–0.55 bits/token worse cross-entropy than the global model. This is expected: each domain model trains on roughly one-quarter of the data available to the global model, reducing its capacity to model general language statistics.

**Interpretation.** The acceptance-overlap gain and the cross-entropy penalty are not contradictory. Acceptance overlap measures distributional alignment with a *domain-conditioned* target. The domain bigram models, despite being weaker general language models, are better aligned with the domain-specific target trigram distributions because they capture domain-characteristic vocabulary and topic patterns. The global bigram model, while a better predictor of text in general, distributes probability mass more uniformly across domains and thus aligns less closely with any single domain-conditioned target.

This distinction is critical: the positive result is mechanism-specific. It supports the claim that domain routing improves acceptance overlap when the target distribution is domain-conditioned, but it does not support the broader claim that domain-specialized drafts are universally superior. In a real speculative-decoding deployment, the cross-entropy penalty could manifest as shorter accepted sequences or more diverse proposals that the target model rejects for reasons not captured by the overlap proxy.

## 4. Limitations

1. **N-gram proxy, not transformer speculative decoding.** The draft and target models are n-gram language models, not neural language models. The acceptance-overlap metric is a proxy for the actual accept/reject step in speculative decoding. Real speculative decoding with transformer models may exhibit different dynamics, including interactions between draft quality, draft length, verification parallelism, and routing latency. This is a toy simulation, not a production validation.

2. **No latency measurement.** The experiment measures acceptance overlap but does not measure wall-clock latency, throughput, or the overhead of the routing step. In production, the router must be fast enough that its latency does not negate the acceptance-rate improvement. The TF-IDF + logistic-regression router used here is lightweight, but its latency profile under production conditions (large batch sizes, long sequences) is untested.

3. **Domain bigrams are weaker conventional LMs.** The per-domain bigram models have worse held-out cross-entropy than the global bigram (Section 3.3). In a real speculative-decoding setting, this could mean that domain drafts propose tokens the target model rejects for reasons beyond domain mismatch—grammatical errors, incoherent continuations, or vocabulary gaps—reducing or eliminating the acceptance-rate advantage observed in the proxy.

4. **Coarse domain granularity.** The four-domain partition is a simplification. Real-world prompts may span multiple domains, fall between domains, or belong to fine-grained subdomains not captured by the router. Misrouting in a finer-grained system could reduce the observed lift.

5. **Oracle target distribution.** The target distribution is an oracle per-domain trigram model, meaning the target's domain conditioning is known exactly. In practice, the target model's domain conditioning is implicit and may not align cleanly with the router's categories. The favorable assumption that the target is cleanly domain-conditioned may overstate the achievable lift.

6. **Small vocabulary and short contexts.** The vocabulary is capped at 5,000 tokens, and contexts are drawn from relatively short newsgroup posts. Scaling to larger vocabularies (30k–100k tokens) and longer contexts may change the magnitude of the observed effects, potentially reducing the overlap advantage of domain-specific models.

7. **Single dataset.** Results are specific to 20 Newsgroups. Generalization to other corpora, languages, or modalities is untested.

8. **No quality evaluation.** The proxy measures distributional overlap but does not evaluate whether the accepted tokens produce coherent, high-quality output. A real speculative-decoding benchmark must assess output quality (e.g., via perplexity or human evaluation) to rule out quality regressions from domain-specialized drafts.

## 6. Conclusion

A lightweight prompt-domain router (TF-IDF + logistic regression, 87.7% accuracy) selecting among four domain-specific bigram draft models yields a +20.7% lift in acceptance overlap over a global bigram draft model on a 4,000-context evaluation of the 20 Newsgroups corpus. The router captures 99.8% of the oracle domain-selection lift. A three-seed robustness sweep confirms stability (mean lift: +20.3%, range: +20.0% to +20.7%).

However, the domain-specific drafts are worse conventional language models than the global draft (0.47–0.55 bits/token higher cross-entropy), and the entire experiment uses n-gram proxy models rather than transformer LLMs. The positive result is specific to acceptance-overlap alignment against a domain-conditioned target distribution and does not imply that domain-specialized drafts are universally better or that production throughput gains will materialize.

The core mechanism—domain routing to improve draft–target distributional alignment—is viable enough under these controlled conditions to warrant a real LLM speculative-decoding benchmark. Such a benchmark should measure accepted tokens per draft step, end-to-end latency, target-model call count, and output quality on mixed-domain prompts with shared target models, global draft models, per-domain draft models or adapters, and the trained router. Only such a benchmark can determine whether the acceptance-overlap improvement observed in this proxy translates to production throughput gains without quality regression.
