## Abstract

We investigate whether a SpecTr-style optimal-transport (OT) verifier can be adapted from flat $k$-candidate speculative decoding to multi-candidate draft trees while preserving the target model distribution. We implement a node-local OT coupling that, at each accepted tree prefix, solves a linear program over a draft child tuple and the target next-token distribution, minimizing rejection probability subject to exact target marginal constraints. In exact finite-vocabulary experiments (vocabulary sizes 5 and 8, tree depths 3–4, candidate counts 2–3), the OT verifier preserves the target sequence distribution to numerical precision (total variation distances on the order of $10^{-16}$) and substantially increases expected accepted tree depth compared to both an exact independent baseline and a greedy feasible coupling. However, the LP variable count scales as $V^{k+1}$, and root-level solve time grows from 0.93 ms at $k{=}1$ to 1,691 ms at $k{=}4$ for $V{=}8$, confirming the exponential-in-$k$ scaling identified in the original SpecTr formulation. The exact LP is therefore not directly viable at full LLM vocabulary scales. Practical deployment would require approximate transport, support truncation with residual correction, or acceptance of near-optimal greedy couplings.

## Introduction

Speculative decoding accelerates autoregressive inference by drafting candidate token sequences from a fast approximate model and verifying them against a target model. The SpecTr framework (Sun et al., 2023) reformulates speculative decoding as an optimal transport problem: given $k$ token-level candidates from a draft distribution and a target next-token distribution, find a coupling that minimizes the probability that the transported token falls outside the candidate set, subject to the constraint that the target marginal is preserved exactly. This guarantees distributional correctness while maximizing acceptance probability.

SpecTr's original analysis focuses on flat $k$-candidate settings and notes that the exact linear program has $V^{k+1}$ variables for vocabulary size $V$, rendering it exponential in $k$. Recent work on tree-based speculative decoding extends the candidate structure from flat sequences to tree-structured drafts, where multiple candidate continuations branch at each position. This raises the question: can a SpecTr-style OT verifier be composed node-locally over a multi-candidate draft tree while preserving the target sequence distribution?

We address this by implementing and testing a node-local OT verifier for draft trees. At each accepted tree prefix, we solve a coupling between the draft child tuple at that node and the target conditional next-token distribution. We compose these local couplings recursively down the tree. We compare against two baselines: an exact independent verifier and a greedy feasible coupling. We evaluate distributional fidelity via total variation distance from the target distribution and practical benefit via expected accepted tree depth.

## Method

### Problem Setting

Consider a draft tree of depth $D$ over a finite vocabulary of size $V$. At each node corresponding to an accepted prefix, the draft model proposes an ordered tuple of $k$ child candidates $Y = (y_1, \ldots, y_k)$. The target model specifies a conditional next-token distribution $P[X = x]$ over $x \in \{1, \ldots, V\}$.

### Node-Local OT Coupling

At each tree node, we solve the following linear program. Let $\pi(x, y)$ denote the coupling between target token $X$ and draft candidate tuple $Y$. The objective is to minimize the probability that the target token is not among the draft candidates:

$$\min_{\pi} \sum_{x \notin \{y_1,\ldots,y_k\}} \pi(x, y)$$

subject to:

- Target marginal: $\sum_{y} \pi(x, y) = P[X = x]$ for all $x$
- Non-negativity: $\pi(x, y) \geq 0$
- Normalization: $\sum_{x,y} \pi(x, y) = 1$

The membership cost structure means that any target token $x$ appearing in the candidate set $Y$ can be transported at zero cost, while tokens outside the set incur unit cost. The LP has $V^{k+1}$ variables (all possible $(x, y_1, \ldots, y_k)$ tuples) and $V$ marginal constraints plus normalization.

### Recursive Composition

The node-local couplings are composed recursively: starting from the root, we sample according to the root coupling. If the sampled token matches a draft candidate, we proceed to the corresponding child node and apply its local coupling, conditioned on the accepted prefix. This recursive procedure defines a distribution over full tree outputs.

### Baselines

We compare three verifiers:

1. **Independent** (`mode=independent`): Exact verification where each candidate is accepted or rejected independently against the target distribution. This preserves the target marginal exactly but does not coordinate across candidates.

2. **Greedy feasible** (`mode=greedy`): A feasible coupling constructed greedily, assigning target probability mass to matching candidates first, then distributing residual mass. This preserves the target marginal but may not minimize rejection probability.

3. **OT LP** (`mode=ot_lp`): The exact optimal transport coupling solving the LP above.

### Distribution Validation

We implement a recursive tree distribution checker that computes the exact output distribution of the full tree verification procedure for each verifier mode, given prefix-conditional target and draft models. This allows direct comparison of the output distribution against the target sequence distribution via total variation (TV) distance.

## Results

### Smoke Test: Vocabulary 5, Depth 4, k=2

| Verifier | TV distance vs. target | Expected accepted nodes / 4 | Mean node accept rate | Total solve time (ms) |
|---|---:|---:|---:|---:|
| Independent | 2.30e-16 | 2.1703 | 0.5341 | 4.43 |
| Greedy feasible | 1.75e-16 | 3.8307 | 0.9588 | 15.45 |
| OT LP | 2.72e-16 | 3.9943 | 0.9987 | 137.24 |

All three verifiers preserve the target distribution to numerical precision (TV distances on the order of $10^{-16}$, consistent with floating-point rounding). The OT LP achieves a mean node accept rate of 0.9987, nearly doubling the expected accepted depth compared to the independent baseline (3.99 vs. 2.17 out of 4). The greedy feasible coupling also performs well (3.83 expected accepted nodes), though not as well as the OT LP.

### Calibration Test: Vocabulary 8, Depth 3, k=3

| Verifier | TV distance vs. target | Expected accepted nodes / 3 | Mean node accept rate | Total solve time (ms) |
|---|---:|---:|---:|---:|
| Independent | 7.81e-16 | 1.4645 | 0.4799 | 58.09 |
| Greedy feasible | 3.09e-16 | 2.9862 | 0.9962 | 98.02 |
| OT LP | 9.43e-16 | 3.0000 | 1.0000 | 1824.02 |

With a larger vocabulary and more candidates, the OT LP achieves a mean node accept rate of 1.0000 and expected accepted depth of 3.0000 (out of 3), meaning every node-level verification succeeds. The greedy feasible coupling is near-optimal (0.9962 mean accept rate). The independent baseline performs substantially worse (0.4799 mean accept rate). Again, all verifiers preserve the target distribution to numerical precision.

### LP Scaling

Root-level LP solve times and variable counts for vocabulary $V = 8$:

| k | Variables | Solve time (ms) |
|---:|---:|---:|
| 1 | 64 | 0.93 |
| 2 | 512 | 1.89 |
| 3 | 4,096 | 25.13 |
| 4 | 32,768 | 1,690.81 |

The variable count follows $V^{k+1}$ as expected. Solve time grows super-polynomially: from under 1 ms at $k{=}1$ to over 1.6 seconds at $k{=}4$ for $V{=}8$. At LLM vocabulary scales ($V \approx 32{,}000$–$128{,}000$), even $k{=}1$ would yield on the order of $10^9$–$10^{10}$ variables, and $k \geq 2$ is infeasible for exact LP.

## Limitations

1. **Exponential scaling in $k$.** The exact LP has $V^{k+1}$ variables. At LLM vocabulary scales, even $k{=}1$ is impractical for the full-vocabulary LP. This is the same fundamental limitation identified in the original SpecTr paper. Our empirical scaling data (0.93 ms at $k{=}1$ to 1,691 ms at $k{=}4$ for $V{=}8$) confirms this concretely.

2. **Small-vocabulary toy simulations only.** All experiments use $V \leq 8$ and $k \leq 3$. These are toy-scale simulations that validate the mathematical correctness of the composition, not the computational feasibility at production scale. No experiments were conducted with real language models or realistic vocabulary sizes.

3. **No production latency evidence.** The solve times reported are for exact LP solutions on tiny vocabularies. We have no evidence about the latency of approximate or truncated variants at production scale, nor about integration with actual model scoring.

4. **Greedy coupling near-optimality may not generalize.** The greedy feasible coupling performed surprisingly well in these experiments (mean accept rate 0.9588–0.9962), approaching the OT LP optimum. However, this near-optimality may depend on the specific draft and target distributions tested and may not hold in general, particularly when draft and target distributions diverge significantly.

5. **Ordered iid draft assumption.** The current implementation assumes draft candidates are drawn iid at each node. Real draft trees may use correlated or conditioned candidate generation strategies, which would change the LP structure.

6. **No approximate transport evaluation.** The SpecTr paper proposes K-sequential and approximate transport methods for large vocabularies. We have not implemented or evaluated these alternatives.

7. **Claim audit status.** The structured claim ledger for this artifact recorded no formal claims and was flagged as `blocked_empty_claims` during audit. The results reported here are drawn directly from run notes and metric files but have not passed a structured claim/evidence audit.

## Conclusion

A SpecTr-style node-local optimal transport verifier can be composed over multi-candidate draft trees and preserves the target sequence distribution to numerical precision in exact finite-vocabulary experiments. The OT verifier substantially improves expected accepted tree depth over an independent baseline and modestly improves over a greedy feasible coupling. These results confirm that the distributional correctness guarantee of SpecTr-style verification extends to the tree setting via node-local composition.

The central practical limitation is the exponential scaling of the exact LP in the candidate count $k$, which renders the exact solution infeasible at LLM vocabulary scales. The near-optimal performance of the greedy feasible coupling in our small-scale tests suggests that practical near-optimal transport may be achievable without exact LP, but this requires further investigation at scale. Viable paths forward include SpecTr-style K-sequential approximate transport, top-p/top-k support truncation with exact residual correction, or direct use of greedy feasible couplings when near-optimality is acceptable.
