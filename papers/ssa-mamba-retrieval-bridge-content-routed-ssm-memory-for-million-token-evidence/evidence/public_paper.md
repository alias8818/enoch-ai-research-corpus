## Abstract

State-space sequence models offer attractive long-context scaling, but fixed-size recurrent state is not by itself an exact retrieval memory. This artifact evaluates a narrow hybrid idea: use an SSM-like stream processor as the sequential substrate while attaching a content-routed external memory keyed by context and evidence key. In a deterministic synthetic retrieval probe, a fixed-capacity FIFO state and a context-blind key router both fail once repeated keys and long spans exceed their assumptions. A context-aware routed bridge preserves exact joint recall at 10k and 100k tokens with 1.0 accuracy, but its 1M-token result depends on memory capacity: underprovisioned routing reaches only 0.4 accuracy across 20 trials, while a larger routed-memory configuration reaches 1.0 accuracy in a single 1M-token trial. The result supports the bridge as a hybrid external-memory architecture, not as evidence that a bounded SSM hidden state alone solves million-token retrieval.

## 1. Question

The motivating question was whether a Mamba/SSM-style long-context system could support million-token evidence retrieval through a new bridge design. The local run deliberately separates three mechanisms:

1. a fixed-capacity recurrent-state proxy (`FIFOState`),
2. a context-blind key router (`KeyOnlyRouter`), and
3. a context-aware external-memory bridge (`ContextAwareBridge`) keyed by `(context, key)`.

This is a mechanism probe, not an end-to-end trained language-model benchmark. The useful test is whether exact retrieval survives repeated keys, growing sequence length, and finite memory capacity.

## 2. Background and motivation

The project decision cites Mamba and subsequent long-context SSM work as motivation for treating content-based retrieval as an unresolved weakness rather than an assumed property of SSM recurrence. The experiment therefore does not try to prove that vanilla SSM state is enough. Instead, it asks whether an explicit content-routed memory layer can preserve exact retrieval behavior in a setting where fixed state and context-blind routing fail.

## 3. Method

The probe script generates synthetic streams containing context/key/value items and then queries whether a model proxy can recover the correct value for the joint `(context, key)` identity. The three model proxies share the same generated task stream and are compared by exact retrieval accuracy.

The reported commands produced:

- `public_evidence/calibration.json`: 1k, 10k, and 100k token lengths with routed-memory capacity of 64 shards x 20,000 entries.
- `public_evidence/million.json`: 1M token length with the same 64 x 20,000 routed capacity.
- `public_evidence/million_one_trial.json`: 1M token length with 128 shards x 40,000 entries.

## 4. Results

### 4.1 Calibration lengths

| Length | Trials | FIFO accuracy | Context-blind accuracy | Context-routed accuracy | Throughput |
|---:|---:|---:|---:|---:|---:|
| 1,000 | 20 | 1.0 | 0.0 | 1.0 | 3.32M items/s |
| 10,000 | 10 | 0.0 | 0.0 | 1.0 | 2.92M items/s |
| 100,000 | 4 | 0.0 | 0.0 | 1.0 | 2.65M items/s |

The fixed FIFO proxy succeeds at 1k tokens but collapses by 10k tokens. The context-blind key router fails at every length, showing that key-only lookup is insufficient when context disambiguation matters. The context-aware bridge maintains exact accuracy through 100k tokens in the calibration run.

### 4.2 Million-token capacity test

At 1M tokens with 64 shards x 20,000 capacity, the context-routed bridge reaches only 0.4 accuracy across 20 trials. FIFO and context-blind routing remain at 0.0. This is the most important negative result: routing does not remove the need for sufficient external memory capacity.

### 4.3 Million-token sufficient-capacity probe

With 128 shards x 40,000 capacity, the context-routed bridge reaches 1.0 accuracy in a single 1M-token trial, with approximately 2.14M updates per second. This supports feasibility of the hybrid mechanism when memory capacity scales with evidence volume, but it is only a one-trial probe and should not be treated as a robust benchmark distribution.

## 5. Interpretation

The result is positive only for the hybrid bridge formulation. The experiment does not support the stronger claim that a bounded SSM hidden state can serve as a million-token exact-retrieval memory. In fact, the fixed-capacity proxy fails exactly where the hybrid bridge becomes useful. The bridge should therefore be understood as an external content-addressed memory design that may pair with an SSM router or stream processor, not as a pure-state replacement for attention or retrieval.

## 6. Limitations

1. **Synthetic proxy task.** The probe uses deterministic synthetic retrieval, not a trained Mamba checkpoint or a real corpus.
2. **No generation quality measurement.** Exact retrieval accuracy is measured directly; no LLM answers are generated or scored.
3. **Capacity dependence.** The 1M-token success requires larger routed capacity. Underprovisioned routing fails partially.
4. **Single high-capacity million-token trial.** The sufficient-capacity 1M result is one trial, useful as a feasibility smoke test but not as a statistical claim.
5. **No latency integration with a real SSM.** The measured throughput is for the Python probe/proxy path, not a production fused SSM plus memory implementation.

## 7. Conclusion

The SSA-Mamba retrieval bridge is viable as a hybrid content-routed memory architecture. The evidence supports a narrow mechanism claim: context-aware external memory can preserve exact joint recall in long synthetic streams where fixed-capacity recurrent state and context-blind routing fail. The evidence also rejects a stronger pure-SSM interpretation: without explicit scalable memory capacity, million-token retrieval is not solved. A useful next step is a real-corpus retrieval replay that measures the capacity/latency tradeoff and tests whether an SSM router can reduce external-index load without degrading recall.
